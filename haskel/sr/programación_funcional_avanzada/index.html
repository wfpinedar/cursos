Programación Funcional Avanzada¶

En este módulo, profundizaremos en las técnicas avanzadas de programación funcional en Haskell. Aprenderás sobre conceptos como programación con continuaciones y currying, que te permitirán escribir código más elegante y eficiente en tus proyectos.

Teoría¶

La programación funcional se basa en el uso de funciones como bloques de construcción fundamentales para crear programas. Esto implica que las funciones son tratadas como valores de primera clase, lo que significa que pueden ser pasadas como argumentos, retornadas como resultados y almacenadas en variables. Además, las funciones puras, que no tienen efectos secundarios y siempre producen el mismo resultado para los mismos argumentos, son altamente valoradas en la programación funcional.

Programación con continuaciones¶

La programación con continuaciones es una técnica avanzada que consiste en pasar una función como argumento a otra función. Esta función, conocida como continuación, se encarga de manejar el resultado de la función a la que se le pasó como argumento. Esto permite escribir código de manera más flexible y expresiva, ya que podemos definir diferentes continuaciones para diferentes situaciones.

Currying¶

Currying es una técnica que consiste en transformar una función que toma múltiples argumentos en una serie de funciones que toman un solo argumento. Esto permite una mayor flexibilidad en la aplicación de funciones y nos permite construir nuevas funciones a partir de otras de manera más sencilla.

Palabras clave¶

  • Programación funcional: Paradigma de programación que se basa en el uso de funciones como bloques de construcción fundamentales para crear programas.
  • Funciones puras: Funciones que no tienen efectos secundarios y siempre producen el mismo resultado para los mismos argumentos.
  • Continuaciones: Funciones que se pasan como argumento a otras funciones para manejar su resultado.
  • Currying: Técnica que consiste en transformar una función que toma múltiples argumentos en una serie de funciones que toman un solo argumento.

Preguntas de repaso¶

  1. ¿Qué es la programación funcional?
  2. ¿Qué son las funciones puras y por qué son valoradas en la programación funcional?
  3. ¿En qué consiste la programación con continuaciones?
  4. ¿Qué es el currying y para qué se utiliza en la programación funcional?

Ejemplos de código en Haskell¶

Programación con continuaciones¶

-- Función que suma dos números y pasa el resultado a una continuación
suma :: Int -> Int -> (Int -> r) -> r
suma x y cont = cont (x + y)

-- Continuación que imprime el resultado
imprimir :: Int -> IO ()
imprimir x = putStrLn ("El resultado es: " ++ show x)

-- Llamada a la función suma con una continuación
main :: IO ()
main = suma 5 3 imprimir -- Imprime "El resultado es: 8"

Currying¶

-- Función que suma dos números
suma :: Int -> Int -> Int
suma x y = x + y

-- Función que aplica un número a la función suma para obtener una nueva función
sumarCinco :: Int -> Int
sumarCinco = suma 5 -- Equivalente a sumarCinco x = suma 5 x

-- Llamada a la función sumarCinco
main :: IO ()
main = print (sumarCinco 3) -- Imprime 8

Ejercicios prácticos¶

  1. Crea una función multiplicar que tome dos números y los multiplique. Luego, crea una continuación que imprima el resultado y llama a la función con los números 4 y 5.
  2. Define una función esPar que tome un número y retorne True si es par y False si es impar. Luego, utiliza esta función en una continuación para imprimir el resultado de determinar si el número 6 es par.
  3. Utiliza currying para crear una función restar que tome dos números y retorne la resta del segundo al primero. Luego, crea una nueva función restarCinco a partir de restar y utilízala para restar 5 a 10.

Consejos y mejores prácticas¶

  • Practica la programación con continuaciones y currying para familiarizarte con estas técnicas.
  • Aprende a utilizar funciones de orden superior para escribir código más elegante y expresivo.
  • Utiliza funciones puras siempre que sea posible para evitar efectos secundarios y hacer tu código más predecible y fácil de mantener.
