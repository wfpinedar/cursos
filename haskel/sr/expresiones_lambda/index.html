Expresiones Lambda¶

Teoría¶

Las expresiones lambda son una característica clave de Haskell que permite escribir funciones de forma concisa y elegante. Una expresión lambda es una función anónima que no tiene un nombre definido y se utiliza para realizar cálculos o transformaciones en los datos. En Haskell, las expresiones lambda se crean utilizando la sintaxis \argumentos -> cuerpo de la función, donde los argumentos son los parámetros de la función y el cuerpo es la expresión que se debe evaluar.

Las expresiones lambda se utilizan principalmente para crear funciones de orden superior, es decir, funciones que toman otras funciones como argumentos o devuelven funciones como resultado. Esto permite una programación más funcional y modular, ya que se pueden crear funciones genéricas que se pueden aplicar a diferentes tipos de datos.

Palabras clave y definiciones¶

  • Lambda: un símbolo (\) utilizado en Haskell para definir una expresión lambda.
  • Función de orden superior: una función que toma otras funciones como argumentos o devuelve funciones como resultado.
  • Parámetros: valores que se pasan a una función para que pueda realizar su cálculo.
  • Expresión: una combinación de valores, operadores y funciones que se evalúa para producir un resultado.
  • Cuerpo: la parte de una función que contiene la expresión que se debe evaluar.

Ejemplos de código¶

  • Una función que suma dos números utilizando una expresión lambda:
sumar = \x y -> x + y
  • Una función de orden superior que toma una función y un valor y devuelve el resultado de aplicar la función al valor:
aplicar :: (a -> b) -> a -> b
aplicar f x = f x

doble = aplicar (\x -> x * 2) 5 -- devuelve 10

Ejercicios prácticos¶

  1. Escribe una función lambda que multiplique dos números.
  2. Crea una función de orden superior que tome una lista de números y una función y aplique la función a cada elemento de la lista.
  3. Escribe una expresión lambda que devuelva el cuadrado de un número.
  4. Utiliza la función map y una expresión lambda para transformar una lista de números en su valor absoluto.
  5. Crea una función de orden superior que tome una lista de cadenas y una función y aplique la función a cada cadena de la lista.

Consejos y mejores prácticas¶

  • Utiliza expresiones lambda en lugar de funciones definidas cuando necesites una función simple y no quieras definir un nombre para ella.
  • Asegúrate de que los argumentos y el cuerpo de la función estén separados por un espacio en blanco después del símbolo \.
  • Utiliza paréntesis para agrupar argumentos y expresiones si es necesario. Por ejemplo: (\x -> (x * x)).
  • Evita usar expresiones lambda demasiado complejas, ya que pueden dificultar la lectura y el mantenimiento del código.
