Introducción a Haskell¶

Haskell es un lenguaje de programación funcional, creado en 1990 por un grupo de investigadores liderado por el matemático Haskell Curry. Se basa en el cálculo lambda, lo que lo convierte en un lenguaje elegante y poderoso para resolver problemas matemáticos y lógicos.

Teoría¶

Haskell se caracteriza por su sistema de tipado estático, lo que significa que todos los tipos de datos son verificados en tiempo de compilación, lo que ayuda a evitar errores comunes en tiempo de ejecución. Además, su paradigma funcional se basa en la evaluación de expresiones y la aplicación de funciones, en lugar de la ejecución de instrucciones secuenciales.

En Haskell, todo es una expresión, incluso las funciones. Esto significa que las funciones pueden ser tratadas como cualquier otro valor, pudiendo ser pasadas como argumentos a otras funciones o ser devueltas como resultado de una función.

Otra característica importante de Haskell es que es un lenguaje puro, lo que significa que no permite efectos secundarios. Esto significa que una función siempre devolverá el mismo resultado para los mismos argumentos, lo que facilita la comprensión y el razonamiento sobre el código.

Palabras clave y definiciones¶

  • Tipado estático: sistema de verificación de tipos en tiempo de compilación.
  • Cálculo lambda: modelo matemático utilizado en la programación funcional.
  • Expresión: cualquier valor o combinación de valores que puede ser evaluado.
  • Función: bloque de código que acepta uno o más argumentos y devuelve un resultado.
  • Lenguaje puro: aquel que no permite efectos secundarios.

Ejemplos de código en Haskell¶

Función que devuelve el cuadrado de un número¶

square :: Int -> Int
square x = x * x

Función que devuelve el factorial de un número¶

factorial :: Int -> Int
factorial 0 = 1
factorial n = n * factorial (n-1)

Ejercicios prácticos¶

  1. Crea una función en Haskell que calcule el área de un círculo, dado su radio como argumento.
  2. Crea una función en Haskell que devuelva la suma de los primeros n números naturales.
  3. Crea una función en Haskell que determine si un número es par o impar.

Consejos o mejores prácticas¶

  • Utiliza nombres de funciones y variables descriptivos.
  • Intenta utilizar funciones de orden superior (aquellas que aceptan funciones como argumentos) en lugar de escribir código repetitivo.
  • Utiliza la recursividad en lugar de bucles para resolver problemas.
  • Prueba tu código con diferentes valores para asegurarte de que funciona correctamente.
