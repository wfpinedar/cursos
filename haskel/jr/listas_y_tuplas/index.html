Listas y Tuplas¶

Introducción¶

En Haskell, al igual que en otros lenguajes de programación, existen diversas estructuras de datos que nos permiten almacenar y manipular información. Entre las más utilizadas se encuentran las listas y las tuplas. Estas estructuras son fundamentales para el desarrollo de programas en Haskell y su correcto uso es esencial para escribir código eficiente y legible.

Listas¶

Las listas son una estructura de datos muy versátil que nos permite almacenar una secuencia ordenada de elementos del mismo tipo. En Haskell, las listas se representan utilizando corchetes y separando cada elemento por comas. Por ejemplo:

numeros = [1, 2, 3, 4, 5] -- lista de números enteros
letras = ['a', 'b', 'c', 'd'] -- lista de caracteres
palabras = ["Hola", "mundo"] -- lista de cadenas de texto

Las listas en Haskell son homogéneas, lo que significa que todos sus elementos deben ser del mismo tipo. Además, son inmutables, lo que significa que una vez creada una lista, no se pueden modificar sus elementos. Sin embargo, es posible crear una nueva lista a partir de otra modificando sus elementos.

Palabras clave¶

  • Listas: estructura de datos en Haskell que permite almacenar una secuencia ordenada de elementos del mismo tipo.
  • Homogéneo: característica de las listas en Haskell que indica que todos sus elementos deben ser del mismo tipo.
  • Inmutable: propiedad de las listas en Haskell que indica que no se pueden modificar sus elementos una vez creada.

Ejemplos de código en Haskell¶

  1. Crear una lista de números enteros y sumar todos sus elementos:
numeros = [1, 2, 3, 4, 5]
suma = sum numeros -- resultado: 15
  1. Obtener el primer elemento de una lista:
numeros = [1, 2, 3, 4, 5]
primerElemento = head numeros -- resultado: 1
  1. Crear una nueva lista a partir de otra modificando sus elementos:
numeros = [1, 2, 3, 4, 5]
nuevosNumeros = map (*2) numeros -- resultado: [2, 4, 6, 8, 10]

Tuplas¶

Las tuplas son otra estructura de datos en Haskell que nos permite almacenar una secuencia de elementos de diferentes tipos. A diferencia de las listas, las tuplas son heterogéneas, lo que significa que pueden contener elementos de distintos tipos. Además, a diferencia de las listas, las tuplas son inmutables, lo que significa que una vez creada una tupla, no se pueden modificar sus elementos.

Las tuplas se representan utilizando paréntesis y separando cada elemento por comas. Por ejemplo:

persona = ("Juan", 25, True) -- tupla con nombre, edad y estado civil

Palabras clave¶

  • Tuplas: estructura de datos en Haskell que permite almacenar una secuencia de elementos de diferentes tipos.
  • Heterogéneo: característica de las tuplas en Haskell que indica que pueden contener elementos de distintos tipos.
  • Inmutable: propiedad de las tuplas en Haskell que indica que no se pueden modificar sus elementos una vez creada.

Preguntas de repaso¶

  1. ¿Qué es una tupla en Haskell?
  2. ¿Cómo se representan las tuplas en Haskell?
  3. ¿Qué significa que una tupla sea heterogénea?
  4. ¿Es posible modificar los elementos de una tupla en Haskell?

Ejemplos de código en Haskell¶

  1. Crear una tupla con información de una persona y obtener su edad:
persona = ("Juan", 25, True)
edad = snd persona -- resultado: 25
  1. Crear una lista de tuplas y obtener el primer elemento de cada una:
personas = [("Juan", 25, True), ("María", 30, False), ("Pedro", 20, True)]
nombres = map fst personas -- resultado: ["Juan", "María", "Pedro"]

Ejercicios prácticos¶

  1. Escribe una función en Haskell que reciba una lista de números enteros y devuelva la suma de todos sus elementos.
  2. Escribe una función en Haskell que reciba una lista de cadenas de texto y devuelva una nueva lista con las cadenas invertidas.
  3. Escribe una función en Haskell que reciba una tupla con información de una persona (nombre, edad, estado civil) y devuelva un mensaje de saludo personalizado.

Consejos y mejores prácticas¶

  • Utiliza listas cuando necesites almacenar una secuencia ordenada de elementos del mismo tipo.
  • Utiliza tuplas cuando necesites almacenar una secuencia de elementos de diferentes tipos.
  • Aprovecha las funciones de orden superior como map y filter para manipular listas y tuplas de forma más eficiente.
  • Recuerda que tanto las listas como las tuplas son inmutables, por lo que siempre se creará una nueva estructura al aplicar una operación sobre ellas.
