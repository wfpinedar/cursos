Programación Orientada a Objetos¶

Explicación teórica¶

La Programación Orientada a Objetos (POO) es un paradigma de programación en el que se modelan los programas como una colección de objetos que interactúan entre sí. Estos objetos tienen características (atributos) y comportamientos (métodos) que definen su estado y cómo se relacionan con otros objetos.

En POO, el código se organiza en clases, que son plantillas para crear objetos. Cada objeto es una instancia de una clase y puede tener sus propios valores para los atributos y comportamientos definidos por la clase.

La POO se basa en cuatro conceptos fundamentales: abstracción, encapsulamiento, herencia y polimorfismo. La abstracción se refiere a la capacidad de modelar objetos en términos de sus características y comportamientos esenciales, mientras que el encapsulamiento se encarga de ocultar los detalles internos de un objeto y solo permitir el acceso a través de métodos definidos.

La herencia permite crear nuevas clases a partir de clases existentes, heredando sus características y comportamientos. Por último, el polimorfismo hace posible que un objeto pueda tener diferentes formas y comportamientos dependiendo del contexto en el que se use.

Palabras clave y su definición¶

  • Objeto: Es una instancia de una clase en POO, que tiene atributos y métodos.
  • Clase: Es una plantilla para crear objetos que define sus atributos y métodos.
  • Atributo: Es una característica de un objeto que define su estado.
  • Método: Es un comportamiento de un objeto que define cómo interactúa con otros objetos.
  • Abstracción: Es el proceso de modelar objetos en términos de sus características y comportamientos esenciales.
  • Encapsulamiento: Es la técnica de ocultar los detalles internos de un objeto y solo permitir el acceso a través de métodos definidos.
  • Herencia: Es la capacidad de crear nuevas clases a partir de clases existentes, heredando sus características y comportamientos.
  • Polimorfismo: Es la capacidad de un objeto de tener diferentes formas y comportamientos dependiendo del contexto en el que se use.

Ejemplos de código en Haskell¶

En Haskell, se pueden utilizar la extensión de tipos de datos para implementar POO. A continuación, se muestra un ejemplo de una clase "Persona" con atributos "nombre" y "edad" y un método "saludar":

{-# LANGUAGE ExistentialQuantification #-}

data Persona = Persona { nombre :: String, edad :: Int }

saludar :: Persona -> String
saludar (Persona nombre _) = "Hola, mi nombre es " ++ nombre

Para crear una instancia de la clase "Persona" y llamar al método "saludar", se puede hacer lo siguiente:

main = do
  let p = Persona "Juan" 25
  putStrLn (saludar p) -- imprimirá "Hola, mi nombre es Juan"

Ejercicios prácticos con instrucciones claras¶

  1. Crea una clase "CuentaBancaria" con los atributos "titular", "saldo" y "tipoCuenta" (ahorro o corriente). Implementa un método "depositar" que reciba un valor y lo sume al saldo de la cuenta.
  2. Crea una instancia de la clase "CuentaBancaria" y realiza una serie de depósitos y retiros utilizando el método "depositar" y otro método "retirar" que reste un valor del saldo.
  3. Crea una subclase "CuentaAhorro" que herede de "CuentaBancaria" y tenga un atributo adicional "interes" y un método "calcularInteres" que calcule el interés a partir del saldo y lo sume al mismo.
  4. Crea una instancia de la subclase "CuentaAhorro" y realiza una serie de depósitos y retiros, además de llamar al método "calcularInteres" para ver cómo afecta al saldo.

Consejos o mejores prácticas¶

  • Utilizar la abstracción y encapsulamiento para mantener el código modular y fácil de mantener.
  • Utilizar la herencia y el polimorfismo para reutilizar código y tener una estructura más escalable.
  • Definir nombres claros y descriptivos para las clases, atributos y métodos.
  • Utilizar la extensión de tipos de datos en Haskell para implementar POO de forma más eficiente.
