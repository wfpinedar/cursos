Programación Concurrente¶

Descripción del Módulo¶

En este módulo, aprenderás a utilizar la programación concurrente en Haskell, un lenguaje funcional puro que ofrece soporte para la ejecución de tareas concurrentes de manera eficiente y segura. Conocerás las herramientas y técnicas necesarias para trabajar con múltiples procesos y hilos de ejecución, así como también la biblioteca de concurrencia proporcionada por Haskell.

1. Explicación teórica¶

La programación concurrente se refiere a la ejecución simultánea de múltiples tareas dentro de un programa. En Haskell, esto se logra a través de la creación de procesos y hilos de ejecución, los cuales pueden ejecutarse de forma paralela o en concurrencia. La diferencia entre estos dos términos radica en cómo se gestionan los recursos del sistema y cómo se coordinan las tareas.

En la programación paralela, los procesos o hilos se ejecutan de forma simultánea en diferentes núcleos de procesamiento, lo que permite un aumento en el rendimiento y la velocidad de ejecución. En cambio, en la programación en concurrencia, los procesos o hilos se ejecutan de forma intercalada en un solo núcleo de procesamiento, compartiendo recursos y coordinándose a través de mecanismos de sincronización.

Haskell ofrece soporte para ambas formas de programación concurrente, lo que permite al programador elegir la mejor opción según las necesidades de su aplicación.

2. Palabras clave y su definición¶

  • Proceso: Es una instancia de un programa en ejecución, que tiene su propio espacio de memoria y recursos asignados.
  • Hilo (Thread): Es una unidad de ejecución ligera dentro de un proceso, que comparte recursos con otros hilos en el mismo proceso.
  • Paralelismo: Se refiere a la ejecución simultánea de tareas en diferentes núcleos de procesamiento.
  • Concurrencia: Se refiere a la ejecución intercalada de tareas en un solo núcleo de procesamiento.
  • Sincronización: Es el mecanismo utilizado para coordinar la ejecución de múltiples procesos o hilos, evitando conflictos y garantizando la consistencia de datos.
  • Biblioteca de concurrencia (Concurrent Haskell): Es una biblioteca de funciones y tipos proporcionada por Haskell para trabajar con procesos y hilos de ejecución.

4. Ejemplos de código en Haskell¶

Creación de un proceso¶

import Control.Concurrent (forkIO)

main = do
  putStrLn "Iniciando el proceso principal"
  forkIO $ do
    putStrLn "Ejecutando proceso en segundo plano"
  putStrLn "Finalizando el proceso principal"

Creación de un hilo¶

import Control.Concurrent (forkIO)
import Control.Concurrent.MVar (newEmptyMVar, takeMVar, putMVar)

main = do
  putStrLn "Iniciando el proceso principal"
  mvar <- newEmptyMVar
  forkIO $ do
    putStrLn "Ejecutando proceso en segundo plano"
    putMVar mvar "Resultado del proceso"
  result <- takeMVar mvar
  putStrLn result
  putStrLn "Finalizando el proceso principal"

5. Ejercicios prácticos con instrucciones claras¶

  1. Crea un programa en Haskell que ejecute dos procesos en paralelo, donde uno imprima "¡Hola!" y el otro imprima "¡Mundo!".
  2. Modifica el programa anterior para que los procesos se ejecuten en concurrencia en lugar de paralelo.
  3. Crea un programa en Haskell que simule una carrera entre dos hilos, donde cada uno avance un paso en cada iteración. El hilo que primero alcance un total de 10 pasos gana la carrera.
  4. Modifica el programa anterior para que los hilos compartan una variable que represente la posición actual en la carrera y se sincronicen para evitar conflictos.

6. Consejos o mejores prácticas¶

  • Utiliza la programación paralela cuando necesites un aumento en el rendimiento y la velocidad de ejecución.
  • Utiliza la programación en concurrencia cuando necesites coordinar tareas y evitar conflictos en el acceso a recursos compartidos.
  • Utiliza la biblioteca de concurrencia de Haskell para aprovechar al máximo las herramientas y funciones disponibles.
  • Utiliza mecanismos de sincronización adecuados para garantizar la consistencia de datos y evitar errores en la ejecución de procesos y hilos.
