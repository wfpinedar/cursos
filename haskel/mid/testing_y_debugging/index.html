Testing y Debugging¶

En el desarrollo de software, es importante asegurar la calidad del código para evitar errores y fallos en la aplicación. Para lograr esto, existen diversas técnicas y herramientas, entre las cuales se encuentran el testing y el debugging.

Explicación teórica¶

El testing consiste en probar el código de una aplicación para verificar que cumpla con los requerimientos y funcione correctamente. Esto se logra mediante la creación de casos de prueba que cubran todas las posibles situaciones que puedan ocurrir en la aplicación. El objetivo del testing es detectar errores y fallos en el código antes de que llegue a producción.

Por otro lado, el debugging es el proceso de encontrar y corregir errores en el código. Esto se logra utilizando herramientas de depuración que permiten examinar el estado del programa en tiempo de ejecución, identificar el origen del error y solucionarlo.

Palabras clave y su definición¶

  • Testing: proceso de probar el código de una aplicación para verificar su correcto funcionamiento.
  • Caso de prueba: conjunto de condiciones y datos de entrada que se utilizan para probar una funcionalidad específica del código.
  • Depuración (debugging): proceso de encontrar y solucionar errores en el código.
  • Herramientas de depuración: programas o utilidades que permiten examinar el estado del programa en tiempo de ejecución para encontrar errores.
  • Pruebas unitarias: pruebas individuales que se realizan en cada componente o unidad de código.
  • Pruebas de integración: pruebas que se realizan en la interacción entre diferentes componentes del código.
  • Pruebas de aceptación: pruebas que se realizan para verificar que la aplicación cumpla con los requerimientos del usuario.

Ejemplos de código en Haskell¶

  1. Ejemplo de una prueba unitaria en Haskell:
-- Función que suma dos números
sumar :: Int -> Int -> Int
sumar x y = x + y

-- Prueba unitaria
main = do
    if sumar 2 3 == 5
        then putStrLn "Prueba aprobada"
        else putStrLn "Prueba fallida"
  1. Ejemplo de una prueba de integración en Haskell:
-- Función que devuelve la suma de los elementos de una lista
sumarLista :: [Int] -> Int
sumarLista [] = 0
sumarLista (x:xs) = x + sumarLista xs

-- Prueba de integración
main = do
    let lista = [1, 2, 3, 4, 5]
    if sumarLista lista == 15
        then putStrLn "Prueba aprobada"
        else putStrLn "Prueba fallida"

Ejercicios prácticos con instrucciones claras¶

  1. Crea una función en Haskell que calcule el factorial de un número.
  2. Realiza pruebas unitarias para comprobar que la función anterior funciona correctamente.
  3. Crea una función que reciba una lista de números y devuelva el número más grande de la lista.
  4. Realiza una prueba de integración para comprobar que la función anterior devuelve el resultado correcto.
  5. Crea una función en Haskell que determine si un número es primo o no.
  6. Realiza una prueba de aceptación para verificar que la función anterior devuelve el resultado correcto para diferentes números.

Consejos o mejores prácticas¶

  • Realizar pruebas de forma regular durante el desarrollo del código.
  • Utilizar herramientas de depuración para encontrar y solucionar errores de forma eficiente.
  • Utilizar diferentes tipos de pruebas para asegurar la calidad del código.
  • Incluir casos de prueba para situaciones extremas y casos límite.
  • Automatizar las pruebas para facilitar su ejecución y detección de errores.
  • Utilizar comentarios y mensajes de error claros en el código para facilitar el debugging.
  • Realizar pruebas en diferentes entornos y condiciones para asegurar la estabilidad y compatibilidad del código.
