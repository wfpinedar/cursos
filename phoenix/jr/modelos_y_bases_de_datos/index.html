Modelos y bases de datos¶

Los modelos son una parte fundamental en el desarrollo de aplicaciones web con Phoenix Framework. Estos nos permiten representar y manipular los datos de nuestra aplicación de una manera estructurada y eficiente. Para poder trabajar con bases de datos en Phoenix, utilizamos Ecto, un módulo que nos proporciona un conjunto de herramientas para interactuar con diferentes sistemas de gestión de bases de datos como PostgreSQL, MySQL, MongoDB, entre otros.

Explicación teórica¶

En Phoenix, los modelos son representados como módulos de Elixir que se encargan de definir la estructura y las relaciones entre los datos de nuestra aplicación. Estos modelos se relacionan directamente con tablas en la base de datos, lo que nos permite realizar operaciones de lectura, escritura, actualización y eliminación de registros.

Para trabajar con modelos en Phoenix, es necesario seguir el patrón de diseño MVC (Modelo-Vista-Controlador), donde los modelos se encargan de interactuar con la base de datos, las vistas se encargan de mostrar la información al usuario y los controladores actúan como intermediarios entre los modelos y las vistas.

Para crear un modelo en Phoenix, utilizamos el generador de modelos mix ecto.gen.model, que nos ayudará a generar el esqueleto del modelo, así como también la migración correspondiente para crear la tabla en la base de datos.

Palabras clave y su definición¶

  • Modelo: Representación de una entidad o concepto en la base de datos, definido como un módulo en Elixir.
  • Ecto: Módulo de Phoenix que nos proporciona herramientas para interactuar con bases de datos.
  • Migración: Archivo que define los cambios en la estructura de la base de datos.
  • MVC: Patrón de diseño que divide la aplicación en tres componentes: Modelo, Vista y Controlador.
  • Relación: Vínculo entre dos o más modelos, que nos permite acceder y manipular datos relacionados.

Ejemplos de código en Phoenix Framework¶

Para crear un modelo en Phoenix, utilizamos el siguiente comando en la terminal:

mix ecto.gen.model Modelo nombre_campo:tipo_dato nombre_campo:tipo_dato

Esto generará dos archivos: el modelo en lib/app_web/models y la migración en priv/repo/migrations. Por ejemplo:

mix ecto.gen.model User name:string email:string

Esto generará un modelo llamado User con dos campos: name de tipo string y email de tipo string.

Para realizar consultas en la base de datos, podemos utilizar el módulo Repo de Ecto. Por ejemplo, para obtener todos los usuarios de la tabla users en la base de datos, utilizamos el siguiente código en un controlador:

def index(conn, _params) do
  users = App.Repo.all(User)
  render(conn, "index.html", users: users)
end

Ejercicios prácticos con instrucciones claras¶

  1. Crea un modelo llamado Post con los campos title de tipo string y content de tipo text.
  2. Crea una migración para agregar un campo author de tipo string a la tabla posts.
  3. Realiza una consulta para obtener todos los posts de un usuario específico.
  4. Crea un modelo Comment con los campos content de tipo text y post_id de tipo integer.
  5. Crea una relación entre los modelos Post y Comment de manera que un post pueda tener varios comentarios.

Consejos o mejores prácticas¶

  • Utiliza nombres descriptivos y en singular para tus modelos.
  • Identifica de antemano las relaciones entre tus modelos para poder definirlas correctamente.
  • Utiliza las migraciones para realizar cambios en la estructura de la base de datos.
  • Utiliza el módulo Repo de Ecto para realizar consultas en la base de datos de manera eficiente.
  • Sigue el patrón de diseño MVC para mantener tu aplicación organizada y fácil de mantener.
