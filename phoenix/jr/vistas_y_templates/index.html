Vistas y templates en Phoenix Framework¶

Explicación teórica¶

Phoenix Framework es un framework web escrito en el lenguaje de programación Elixir, que sigue el patrón de arquitectura MVC (Modelo-Vista-Controlador). Las vistas y templates son una parte fundamental de este patrón, ya que se encargan de renderizar la interfaz de usuario para que sea visualizada por el usuario final.

Una vista en Phoenix es una función que se encarga de recibir los datos del controlador y generar una respuesta en algún formato, como HTML, JSON o XML. Esta respuesta es enviada al navegador del cliente para que pueda ser visualizada. Por otro lado, un template es un archivo que contiene el código HTML y Elixir necesario para generar una vista. En otras palabras, las vistas utilizan los templates para renderizar el contenido de la página.

Palabras clave y su definición¶

  • Vista: Función en Phoenix encargada de recibir los datos del controlador y generar una respuesta en algún formato.
  • Template: Archivo que contiene el código HTML y Elixir necesario para generar una vista.
  • Renderizar: Proceso de generar una respuesta a partir de una vista y un template.
  • Interfaz de usuario: Parte de la aplicación que interactúa con el usuario final.
  • MVC: Patrón de arquitectura de software que divide una aplicación en tres componentes: Modelo, Vista y Controlador.

Ejemplos de código en Phoenix Framework¶

Vista básica en Phoenix¶

defmodule MyApp.Web.UserView do
  use MyApp.Web, :view

  def index(conn, _params) do
    users = Repo.all(User)
    render(conn, "index.html", users: users)
  end
end

En este ejemplo, se define una vista básica en la que se obtienen todos los usuarios de la base de datos y se los pasa como parámetro al template "index.html" para ser renderizado.

Template básico en Phoenix¶

<!-- index.html.eex -->
<h1>Lista de usuarios</h1>

<ul>
  <%= for user <- @users do %>
    <li><%= user.name %></li>
  <% end %>
</ul>

En este ejemplo, se utiliza el lenguaje de plantillas Elixir para generar una lista con los nombres de los usuarios obtenidos en la vista.

Ejercicios prácticos con instrucciones claras¶

  1. Crea una nueva vista en Phoenix llamada "ProductView".
  2. Define una acción "index" en la vista que reciba todos los productos de la base de datos y los pase como parámetro al template "index.html".
  3. Crea un nuevo template llamado "index.html" en la carpeta "product" de la vista.
  4. Utiliza el lenguaje de plantillas Elixir para generar una tabla con los nombres y precios de los productos obtenidos en la vista.
  5. Crea una acción "show" en la vista que reciba un producto específico y lo pase como parámetro al template "show.html".
  6. Crea un nuevo template llamado "show.html" en la carpeta "product" de la vista.
  7. Utiliza el lenguaje de plantillas Elixir para mostrar los detalles del producto recibido en la vista.

Consejos o mejores prácticas¶

  • Mantener las vistas y templates lo más simples y limpios posible.
  • Utilizar el lenguaje de plantillas Elixir de forma eficiente para evitar código redundante.
  • Utilizar la función "render" en las vistas para pasar los datos al template de forma clara y ordenada.
  • Organizar las vistas y templates en carpetas y subcarpetas según la funcionalidad que cumplan en la aplicación.
  • Utilizar nombres descriptivos para las vistas y templates para facilitar su mantenimiento y comprensión en el futuro.
