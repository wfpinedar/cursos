Creación de un proyecto básico en Phoenix¶

El Phoenix Framework es un framework web de código abierto basado en el lenguaje de programación Elixir y que sigue el patrón de arquitectura Modelo-Vista-Controlador (MVC). Este framework es ideal para el desarrollo de aplicaciones web rápidas y escalables, gracias a su enfoque en la concurrencia y el rendimiento.

Explicación teórica¶

Para crear un proyecto básico en Phoenix, primero debemos tener instalado Elixir y el Phoenix Framework en nuestro sistema. Una vez que tengamos estos requisitos, podemos seguir los siguientes pasos:

  1. Abrir una terminal y navegar hasta el directorio en el que queramos crear nuestro proyecto.
  2. Ejecutar el comando mix phx.new nombre_del_proyecto para crear un nuevo proyecto en Phoenix. Este comando generará una estructura de directorios y archivos para nuestro proyecto.
  3. Navegar hasta el directorio del proyecto (cd nombre_del_proyecto) y ejecutar mix deps.get para instalar las dependencias del proyecto.
  4. Luego, ejecutar mix ecto.create para crear la base de datos para nuestro proyecto.
  5. Finalmente, ejecutar mix phx.server para correr el servidor local y visualizar nuestro proyecto en el navegador.

Palabras clave y su definición¶

  • Phoenix Framework: Framework web de código abierto basado en Elixir que sigue el patrón MVC.
  • Elixir: Lenguaje de programación funcional y dinámico que se ejecuta en la máquina virtual de Erlang (BEAM).
  • MVC: Patrón de arquitectura de software que separa la aplicación en modelos, vistas y controladores.
  • Dependencias: Librerías externas necesarias para el funcionamiento de un proyecto.
  • Base de datos: Conjunto estructurado de datos que se almacena en un sistema de gestión de bases de datos.
  • Servidor local: Servidor que se ejecuta en nuestra propia computadora para acceder al proyecto en desarrollo en el navegador.

Ejemplos de código en Phoenix Framework¶

Creación de un controlador¶

defmodule MiProyectoWeb.UserController do
  use MiProyectoWeb, :controller

  def index(conn, _params) do
    users = Repo.all(User)
    render(conn, "index.html", users: users)
  end
end

Creación de una vista¶

defmodule MiProyectoWeb.UserView do
  use MiProyectoWeb, :view

  def render("index.html", %{users: users}) do
    ~H"""
    <ul>
      <%= for user <- users do %>
        <li><%= user.name %></li>
      <% end %>
    </ul>
    """
  end
end

Ejercicios prácticos con instrucciones claras¶

  1. Crea un nuevo proyecto en Phoenix llamado "mi_proyecto".
  2. Agrega una vista y un controlador para mostrar una lista de usuarios en la página de inicio.
  3. Crea un modelo para almacenar información de usuarios en una base de datos.
  4. Agrega una función en el controlador para crear un nuevo usuario y guardarlo en la base de datos.
  5. Actualiza la vista para mostrar la lista de usuarios actualizada después de crear uno nuevo.
  6. Ejecuta el servidor local y comprueba que todo funcione correctamente en el navegador.

Consejos o mejores prácticas¶

  • Sigue la convención de nombres de Phoenix Framework para mantener una estructura clara y consistente en tu proyecto.
  • Utiliza las migraciones de Ecto para manejar los cambios en la estructura de la base de datos.
  • Aprovecha las ventajas de la concurrencia y el rendimiento de Elixir al desarrollar tu proyecto en Phoenix.
  • Utiliza Phoenix LiveView para agregar interactividad a tu aplicación web en tiempo real.
