Instalación y configuración de Phoenix Framework¶

Explicación teórica¶

Phoenix Framework es un framework web de código abierto escrito en el lenguaje de programación Elixir. Está diseñado para crear aplicaciones web escalables y de alto rendimiento utilizando el patrón de arquitectura Modelo-Vista-Controlador (MVC). Phoenix se basa en el lenguaje de programación funcional Elixir y se ejecuta en la máquina virtual de Erlang (BEAM), lo que le brinda una gran estabilidad y tolerancia a fallos.

Phoenix Framework proporciona una estructura sólida y un conjunto de herramientas para el desarrollo de aplicaciones web, incluyendo un enrutador de URL, un sistema de plantillas, un ORM (Object Relational Mapper) para interactuar con bases de datos, y un sistema de autenticación. También es altamente extensible y permite integrar fácilmente otras herramientas y librerías.

Palabras clave y su definición¶

  • Phoenix Framework: Framework web de código abierto basado en Elixir y la máquina virtual de Erlang (BEAM).
  • Elixir: Lenguaje de programación funcional que se ejecuta en la máquina virtual de Erlang (BEAM).
  • MVC: Patrón de arquitectura Modelo-Vista-Controlador utilizado en el desarrollo de aplicaciones web.
  • ORM: Object Relational Mapper, herramienta que permite interactuar con bases de datos utilizando objetos en lugar de consultas SQL.
  • Enrutador de URL: Sistema que se encarga de dirigir las solicitudes HTTP a las acciones correspondientes en la aplicación.
  • Sistema de plantillas: Herramienta que permite generar contenido HTML dinámico utilizando marcado y lenguaje de programación.
  • Autenticación: Proceso de verificación de la identidad de un usuario para acceder a una aplicación o sistema.

Ejemplos de código en Phoenix Framework¶

  1. Creación de una ruta en el enrutador de URL:
get "/users", UserController, :index
  1. Creación de una acción en el controlador de usuarios:
def index(conn, _params) do
  users = Repo.all(User)
  render(conn, "index.html", users: users)
end
  1. Utilización de una plantilla para mostrar los usuarios en una vista:
<%= for user <- @users do %>
  <p><%= user.name %></p>
<% end %>

Ejercicios prácticos con instrucciones claras¶

  1. Instalación de Phoenix Framework:

  2. Descargar e instalar Elixir y Erlang en tu sistema.

  3. Abrir una terminal y ejecutar el comando mix archive.install hex phx_new para instalar el generador de proyectos de Phoenix.
  4. Crear un nuevo proyecto con el comando mix phx.new my_app y seguir las instrucciones para configurar la base de datos y otras opciones.
  5. Ejecutar mix deps.get para instalar las dependencias del proyecto.
  6. Ejecutar mix ecto.create para crear la base de datos.

  7. Ejecutar mix phx.server para iniciar el servidor web.

  8. Creación de una ruta y una acción en el controlador:

  9. En el archivo lib/my_app_web/router.ex, agregar la siguiente línea dentro del bloque scope "/api", MyAppWeb do:

get "/products", ProductController, :index
  • En el archivo lib/my_app_web/controllers/product_controller.ex, agregar la siguiente acción:
def index(conn, _params) do
  products = Repo.all(Product)
  render(conn, "index.json", products: products)
end

  1. Utilización de una plantilla para mostrar los productos en una vista:

  2. En el archivo lib/my_app_web/templates/product/index.html.eex, agregar el siguiente código:

<%= for product <- @products do %>
  <p><%= product.name %></p>
<% end %>

  1. Creación de un modelo y migración para la tabla de productos:

  2. Ejecutar mix ecto.gen.migration create_products para generar el archivo de migración.

  3. En el archivo generado, agregar las columnas deseadas a la tabla de productos.
  4. Ejecutar mix ecto.migrate para aplicar la migración a la base de datos.
  5. En el archivo lib/my_app/product.ex, agregar las especificaciones del modelo de producto.

Consejos o mejores prácticas¶

  • Familiarizarse con el lenguaje de programación Elixir antes de utilizar Phoenix Framework.
  • Utilizar las convenciones y estructuras recomendadas por Phoenix para mantener una aplicación bien organizada y fácil de mantener.
  • Aprovechar las funcionalidades de enrutamiento y plantillas para facilitar la creación de rutas y vistas en la aplicación.
  • Utilizar librerías y herramientas recomendadas por la comunidad para extender las funcionalidades de Phoenix.
  • Realizar pruebas unitarias y de integración para asegurar el correcto funcionamiento de la aplicación.
