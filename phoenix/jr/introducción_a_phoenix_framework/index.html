Introducción a Phoenix Framework¶

Descripción del módulo¶

Phoenix Framework es un framework de desarrollo web de código abierto basado en el lenguaje de programación Elixir. Fue creado por Chris McCord en 2014 y está inspirado en el framework Ruby on Rails, aunque con algunas diferencias en su arquitectura y filosofía.

En este módulo, exploraremos la historia y características de Phoenix Framework, así como también su uso y ventajas para el desarrollo de aplicaciones web.

Explicación teórica¶

Phoenix Framework se basa en el patrón de arquitectura Modelo-Vista-Controlador (MVC), el cual se utiliza para separar la lógica de negocio, la presentación de datos y la interacción con el usuario en diferentes capas. Esto permite una mayor modularidad, escalabilidad y mantenibilidad en el código de una aplicación web.

Una de las principales características de Phoenix Framework es su enfoque en la concurrencia y el rendimiento. Está construido sobre la plataforma de programación funcional de Elixir, que se ejecuta en la máquina virtual de Erlang (BEAM). Esto permite a Phoenix utilizar el modelo de procesos ligeros (lightweight processes) de Erlang para manejar grandes cantidades de solicitudes concurrentes, sin afectar el rendimiento de la aplicación.

Otra característica importante de Phoenix Framework es su sistema de enrutamiento dinámico, que permite crear rutas y controladores de manera sencilla y flexible. Además, cuenta con un sistema de vistas que utiliza el lenguaje de plantillas EEx para generar HTML de forma eficiente.

Palabras clave y definiciones¶

  • Framework: Es un conjunto de herramientas, librerías y patrones de diseño que facilitan el desarrollo de aplicaciones web.
  • Elixir: Lenguaje de programación funcional diseñado para construir aplicaciones escalables y tolerantes a fallos.
  • MVC: Patrón de arquitectura que separa la lógica de negocio, la presentación de datos y la interacción con el usuario en diferentes capas.
  • Concurrencia: Capacidad de un sistema para realizar múltiples tareas al mismo tiempo.
  • Rendimiento: Medida de la eficiencia con la que una aplicación realiza sus tareas.
  • Procesos ligeros: Unidad de ejecución independiente y liviana en la máquina virtual de Erlang.
  • Erlang: Lenguaje de programación funcional utilizado en la construcción de sistemas distribuidos y tolerantes a fallos.
  • Sistema de enrutamiento: Mecanismo que permite asociar una URL a un controlador y una acción en una aplicación web.
  • Controlador: Componente que maneja las solicitudes HTTP recibidas y es responsable de coordinar la lógica de negocio.
  • Vistas: Componentes que generan la presentación de datos en una aplicación web.
  • EEx: Lenguaje de plantillas basado en Elixir utilizado para generar HTML de forma dinámica.

Ejemplos de código en Phoenix Framework¶

A continuación, se muestra un ejemplo de un controlador en Phoenix Framework que maneja una solicitud GET para obtener todos los usuarios de una aplicación:

defmodule MyApp.UserController do
  use MyApp.Web, :controller

  def index(conn, _params) do
    users = MyApp.Repo.all(MyApp.User)
    render(conn, "index.html", users: users)
  end
end

Este controlador utiliza la función MyApp.Repo.all/1 para consultar todos los usuarios de la base de datos y luego renderiza la vista index.html con los datos obtenidos.

Ejercicios prácticos¶

  1. Crea un nuevo proyecto en Phoenix Framework utilizando el comando mix phx.new my_app.
  2. Crea un modelo y migración de base de datos para almacenar información de libros.
  3. Crea un controlador y sus correspondientes rutas para gestionar la creación y visualización de libros.
  4. Crea una vista que muestre una lista de todos los libros almacenados en la base de datos.
  5. Agrega una funcionalidad para que los usuarios puedan agregar comentarios a los libros.
  6. Implementa una autenticación de usuario utilizando el módulo Comeonin y el sistema de autenticación de Phoenix.
  7. Agrega una funcionalidad de búsqueda para encontrar libros por título o autor.
  8. Crea un sistema de paginación para mostrar una cantidad limitada de libros por página.
  9. Agrega la posibilidad de editar y eliminar libros existentes.
  10. Implementa pruebas automatizadas para el controlador y modelos creados.

Consejos o mejores prácticas¶

  • Utiliza el generador de Phoenix Framework mix phx.gen.html para crear modelos, migraciones, controladores y vistas de manera rápida y sencilla.
  • Sigue las convenciones de nombres y estructura de carpetas de Phoenix Framework para una mejor organización y legibilidad del código.
  • Utiliza el sistema de migraciones de Ecto para realizar cambios en la estructura de la base de datos de forma segura y controlada.
  • Aprovecha las ventajas de la programación funcional en Elixir para escribir un código más legible, mantenible y escalable.
  • Utiliza el módulo Plug para agregar middlewares y funcionalidades adicionales a tu aplicación.
