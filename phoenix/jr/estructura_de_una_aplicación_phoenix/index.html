Estructura de una aplicación Phoenix¶

La estructura de una aplicación Phoenix es uno de los aspectos más importantes a entender para poder desarrollar de manera efectiva con este framework. Phoenix sigue una estructura de convención sobre configuración, lo que significa que hay una manera predeterminada de organizar los archivos y carpetas de una aplicación Phoenix, aunque se pueden realizar cambios si es necesario.

Explicación teórica¶

La estructura de una aplicación Phoenix se basa en el patrón de diseño Modelo-Vista-Controlador (MVC), que separa la lógica de la aplicación en tres partes: el modelo, la vista y el controlador. El modelo se encarga de interactuar con la base de datos y manejar los datos de la aplicación, mientras que la vista se encarga de mostrar la información al usuario. El controlador actúa como intermediario entre el modelo y la vista, procesando las solicitudes del usuario y enviando los datos necesarios a la vista.

Además de los tres componentes principales, una aplicación Phoenix también cuenta con otros directorios y archivos importantes, como las rutas, los modelos de base de datos, las migraciones y los controladores.

Palabras clave y su definición¶

  • MVC: Modelo-Vista-Controlador, un patrón de diseño de software que separa la lógica de una aplicación en tres partes principales: el modelo, la vista y el controlador.
  • Rutas: Definen las URL y las acciones correspondientes en el controlador para manejar las solicitudes del usuario.
  • Modelos de base de datos: Representan las entidades de la aplicación y se encargan de interactuar con la base de datos.
  • Migraciones: Archivos que permiten realizar cambios en la estructura de la base de datos.
  • Controladores: Se encargan de procesar las solicitudes del usuario y enviar los datos necesarios a la vista.
  • Vistas: Son las encargadas de mostrar la información al usuario, generalmente en formato HTML.
  • Ecto: Biblioteca de mapeo objeto-relacional utilizada en Phoenix para interactuar con la base de datos.

Ejemplos de código en Phoenix Framework¶

  1. Definición de una ruta en el archivo router.ex:
get "/users", UserController, :index
  1. Creación de un modelo de base de datos en Phoenix:
defmodule MyApp.User do
  use Ecto.Schema
  import Ecto.Changeset

  schema "users" do
    field :name, :string
    field :email, :string
    field :age, :integer
  end

  def changeset(user, params \\ %{}) do
    user
    |> cast(params, [:name, :email, :age])
    |> validate_required([:name, :email, :age])
    |> validate_format(:email, ~r/@/)
  end
end
  1. Creación de una migración en Phoenix:
defmodule MyApp.Repo.Migrations.CreateUsers do
  use Ecto.Migration

  def change do
    create table(:users) do
      add :name, :string
      add :email, :string
      add :age, :integer

      timestamps()
    end
  end
end

Ejercicios prácticos con instrucciones claras¶

  1. Crea una nueva aplicación Phoenix utilizando el comando mix phx.new my_app.
  2. Agrega una nueva ruta en el archivo router.ex que maneje la solicitud GET /posts.
  3. Crea un modelo de base de datos para la entidad "post" con los campos title, content y author.
  4. Agrega una migración para crear la tabla correspondiente en la base de datos.
  5. Crea un controlador para manejar las solicitudes relacionadas con los posts.
  6. En el controlador, define una acción index que obtenga todos los posts de la base de datos y los envíe a una vista para mostrarlos.
  7. Crea una vista para mostrar los posts en formato HTML.
  8. Agrega un enlace en la página principal de la aplicación que redirija al usuario a la página de posts.
  9. Ejecuta la aplicación y comprueba que los posts se muestren correctamente en la vista.

Consejos o mejores prácticas¶

  1. Sigue la convención sobre configuración de Phoenix y mantén la estructura de la aplicación lo más limpia y organizada posible.
  2. Utiliza nombres descriptivos para las rutas, modelos, controladores y vistas.
  3. Divide la lógica de la aplicación en pequeñas acciones en lugar de crear controladores con demasiadas acciones.
  4. Utiliza migraciones para realizar cambios en la estructura de la base de datos y evita modificarla manualmente.
  5. Investiga sobre los diferentes generadores y herramientas disponibles en Phoenix para agilizar el desarrollo.
  6. Utiliza Ecto para realizar consultas a la base de datos en lugar de escribir consultas SQL directamente.
  7. Utiliza el archivo config.exs para configurar opciones globales de la aplicación.
  8. Lee la documentación oficial de Phoenix para aprender más sobre la estructura y las mejores prácticas de desarrollo.
