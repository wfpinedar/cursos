Phoenix sin servidor¶

Explicación teórica¶

Phoenix Framework es un framework web de código abierto que se basa en el lenguaje de programación Elixir y está diseñado para construir aplicaciones web escalables y de alto rendimiento. Una de las principales ventajas de Phoenix es su capacidad para manejar una gran cantidad de solicitudes simultáneas de forma eficiente. Sin embargo, al desplegar una aplicación Phoenix, es común utilizar un servidor web como Apache o Nginx para manejar las solicitudes entrantes. Sin embargo, existen opciones para desplegar aplicaciones Phoenix sin la necesidad de un servidor.

Una de las formas de hacerlo es utilizando un servicio de computación en la nube como AWS Lambda. AWS Lambda es un servicio de computación sin servidor que permite ejecutar código sin necesidad de administrar servidores. Con AWS Lambda, podemos desplegar nuestra aplicación Phoenix en un entorno sin servidor, lo que significa que no necesitamos preocuparnos por configurar y mantener un servidor web.

Palabras clave y su definición¶

  • Phoenix Framework: Es un framework web de código abierto basado en el lenguaje de programación Elixir.
  • AWS Lambda: Es un servicio de computación en la nube que permite ejecutar código sin necesidad de administrar servidores.
  • Sin servidor: Se refiere a la capacidad de ejecutar aplicaciones sin la necesidad de configurar y mantener un servidor web.
  • Computación en la nube: Se refiere al uso de recursos informáticos, como servidores, almacenamiento y aplicaciones, a través de internet en lugar de tenerlos localmente en un ordenador personal o servidor.

Ejemplos de código en Phoenix Framework¶

Para desplegar una aplicación Phoenix en AWS Lambda, necesitamos realizar los siguientes pasos:

  1. Crear un archivo de configuración lambda.exs en el directorio config de nuestra aplicación Phoenix.
  2. En este archivo, definir la función MyAppWeb.Lambda.handler que será el punto de entrada para nuestra aplicación.
  3. En la función handler, debemos crear una instancia de nuestra aplicación Phoenix y llamar al método run para iniciarla.
  4. Una vez que nuestra aplicación se inicie, debemos definir una función event que recibirá las solicitudes entrantes y las procesará utilizando los controladores y enrutadores de nuestra aplicación Phoenix.
  5. Por último, debemos configurar nuestra aplicación para que utilice el adaptador :cowboy y el puerto 0 para que pueda ser desplegada en AWS Lambda.

Un ejemplo de código para el archivo lambda.exs sería el siguiente:

defmodule MyAppWeb.Lambda do
  def handler(event, context) do
    {:ok, _} = Application.ensure_all_started(:my_app_web)
    {:ok, _} = MyAppWeb.Endpoint.run([])
    event(MyAppWeb.Endpoint, event, context)
  end
end

Ejercicios prácticos con instrucciones claras¶

  1. Crea un nuevo proyecto Phoenix utilizando el comando mix phx.new my_app.
  2. En el directorio config de tu proyecto, crea un archivo llamado lambda.exs.
  3. En este archivo, define la función MyAppWeb.Lambda.handler como se muestra en el ejemplo de código anterior.
  4. Configura tu aplicación Phoenix para utilizar el adaptador :cowboy y el puerto 0 en el archivo config/prod.exs.
  5. Crea un controlador y un enrutador en tu aplicación Phoenix.
  6. En el controlador, define una acción que responda con un mensaje de "Hola mundo!".
  7. En el enrutador, define una ruta para esa acción.
  8. Despliega tu aplicación Phoenix en AWS Lambda utilizando el comando mix phx.server y asegúrate de que funciona correctamente.
  9. Haz una solicitud HTTP a la URL proporcionada por AWS Lambda y verifica que recibes el mensaje "Hola mundo!" en respuesta.
  10. ¡Felicidades, has desplegado con éxito tu aplicación Phoenix sin servidor!

Consejos o mejores prácticas¶

  • Al desplegar una aplicación Phoenix en AWS Lambda, es importante tener en cuenta el límite de tiempo de ejecución de 15 minutos. Si una solicitud requiere más tiempo para procesarse, es posible que AWS Lambda la interrumpa.
  • Utiliza el adaptador :cowboy en lugar de :plug_cowboy cuando despliegues tu aplicación en AWS Lambda, ya que el primero es más adecuado para entornos sin servidor.
  • Utiliza una herramienta de despliegue automatizado como Distillery para facilitar el proceso de despliegue en AWS Lambda.
  • Ten en cuenta que AWS Lambda cobra por el tiempo de ejecución y por el número de solicitudes, por lo que es importante optimizar tu código para que se ejecute de forma eficiente.
  • Si tu aplicación Phoenix utiliza una base de datos, considera utilizar una base de datos sin servidor como Amazon DynamoDB en lugar de una base de datos tradicional, ya que es más adecuada para entornos sin servidor.
