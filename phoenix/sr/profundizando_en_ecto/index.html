Profundizando en Ecto¶

Explicación teórica¶

Ecto es una librería de mapeo de objetos relacional (ORM) utilizada en el framework Phoenix para interactuar con la base de datos. Su función principal es facilitar la manipulación y consulta de datos en la base de datos, evitando la necesidad de escribir consultas SQL directamente.

Ecto se basa en tres conceptos fundamentales: modelos, cambiosets y repositorios. Los modelos representan una tabla en la base de datos y sus relaciones con otras tablas. Los cambiosets son estructuras que se utilizan para validar y manipular los datos antes de ser guardados en la base de datos. Los repositorios son responsables de ejecutar consultas y almacenar los resultados en el modelo correspondiente.

Palabras clave y su definición¶

  • ORM: siglas de Object-Relational Mapping, que se refiere a la técnica de mapear objetos de una aplicación a tablas de una base de datos relacional.
  • Modelo: representa una tabla en la base de datos y sus relaciones con otras tablas.
  • Cambioset: estructura utilizada para validar y manipular los datos antes de ser guardados en la base de datos.
  • Repositorio: responsable de ejecutar consultas y almacenar los resultados en el modelo correspondiente.

Ejemplos de código en Phoenix Framework¶

Ejemplo de definición de un modelo en Ecto:

defmodule Post do
  use Ecto.Schema
  import Ecto.Changeset

  schema "posts" do
    field :title, :string
    field :content, :string
    has_many :comments, Comment
  end

  def changeset(post, attrs) do
    post
    |> cast(attrs, [:title, :content])
    |> validate_required([:title, :content])
    |> validate_length(:content, min: 10)
  end
end

Ejemplo de consulta utilizando Ecto:

# Buscar todos los posts con más de 10 comentarios y ordenarlos por fecha de creación
posts = Repo.all(from p in Post,
  join: c in assoc(p, :comments),
  where: c.id > 10,
  order_by: [desc: p.inserted_at]
)

Ejercicios prácticos con instrucciones claras¶

  1. Crea un modelo de Ecto llamado "User" con los campos "name", "email" y "password".
  2. Implementa un cambioset para validar que el campo "email" sea único.
  3. Crea una consulta utilizando Ecto para obtener los usuarios ordenados alfabéticamente por su nombre.
  4. Crea una migración de base de datos para agregar un campo "role" al modelo "User".
  5. Modifica el cambioset para validar que el campo "role" sea uno de los valores permitidos: "admin", "editor" o "viewer".

Consejos o mejores prácticas¶

  1. Utiliza alias para acortar el nombre de los módulos de Ecto en tus consultas y cambiosets.
  2. Aprovecha las asociaciones en tus modelos para simplificar tus consultas.
  3. Utiliza transacciones para asegurar la consistencia de tus datos en operaciones complejas.
  4. Utiliza Ecto.Multi para realizar operaciones en lotes de forma eficiente.
  5. Utiliza Ecto.Repo.insert_all para insertar múltiples registros en una sola consulta.
  6. Utiliza Ecto.Query.preload para cargar datos relacionados en una sola consulta.
  7. Utiliza Ecto.Query.paginate para paginar tus resultados de forma eficiente.
  8. Siempre utiliza Ecto.Changeset para validar y manipular los datos antes de guardarlos en la base de datos.
  9. Utiliza Ecto.Schema para definir tus modelos y sus relaciones de manera clara y sencilla.
  10. Sigue las convenciones de nomenclatura de Ecto para evitar errores y facilitar el mantenimiento de tu código.
