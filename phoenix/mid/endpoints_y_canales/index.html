Endpoints y canales en Phoenix Framework¶

Descripción del módulo¶

En este módulo aprenderás a trabajar con endpoints y canales en Phoenix Framework, una herramienta de desarrollo web que utiliza el lenguaje de programación Elixir. Los endpoints y canales son fundamentales para construir aplicaciones en tiempo real, ya que permiten la comunicación entre el cliente y el servidor de forma bidireccional y en tiempo real.

Explicación teórica¶

Endpoints¶

Un endpoint en Phoenix Framework es un punto de acceso a una aplicación web. Es el encargado de recibir las peticiones HTTP del cliente y enviar las respuestas correspondientes. Cada endpoint tiene una ruta, que es la dirección a la que se envía la petición y un controlador, que es el encargado de manejar la lógica de la petición y generar la respuesta.

En Phoenix, los endpoints se definen en el archivo endpoint.ex dentro de la carpeta lib/nombre_de_la_app_web/. Por lo general, una aplicación Phoenix tiene varios endpoints, cada uno para manejar diferentes tipos de peticiones.

Canales¶

Los canales en Phoenix Framework son una forma de establecer una conexión bidireccional y en tiempo real entre el cliente y el servidor. Se utilizan principalmente para construir aplicaciones en tiempo real como chats, juegos en línea, entre otros.

Los canales están basados en el protocolo de websockets, que permite una comunicación bidireccional y en tiempo real entre el cliente y el servidor. En Phoenix, los canales se definen en el archivo socket.ex dentro de la carpeta lib/nombre_de_la_app_web/channels/. Cada canal tiene un identificador único y una función handle_in que se encarga de manejar los eventos recibidos del cliente.

Palabras clave y su definición¶

  • Endpoint: Punto de acceso a una aplicación web que recibe peticiones HTTP del cliente y envía respuestas.
  • Controlador: Función que maneja la lógica de una petición en un endpoint.
  • Ruta: Dirección a la que se envía una petición HTTP.
  • Canal: Conexión bidireccional y en tiempo real entre el cliente y el servidor.
  • Protocolo de websockets: Protocolo de comunicación que permite una conexión bidireccional y en tiempo real entre el cliente y el servidor.

Ejemplos de código en Phoenix Framework¶

Definición de un endpoint¶

# lib/nombre_de_la_app_web/endpoint.ex
defmodule NombreDeLaAppWeb.Endpoint do
  use Phoenix.Endpoint, otp_app: :nombre_de_la_app_web

  # Ruta del endpoint
  socket "/socket", NombreDeLaAppWeb.UserSocket

  # Configuración de la base de datos
  # ...

  # Configuración de las vistas
  # ...

  # Configuración del pipeline de la aplicación
  # ...
end

Definición de un canal¶

# lib/nombre_de_la_app_web/channels/user_socket.ex
defmodule NombreDeLaAppWeb.UserSocket do
  use Phoenix.Socket

  # Identificador único del canal
  channel "user:*", NombreDeLaAppWeb.UserChannel

  # Función para manejar eventos recibidos del cliente
  def handle_in("nuevo_mensaje", %{"usuario" => usuario, "mensaje" => mensaje}, socket) do
    broadcast! socket, "nuevo_mensaje", %{usuario: usuario, mensaje: mensaje}
    {:reply, {:ok, %{mensaje: "Mensaje enviado con éxito"}}, socket}
  end
end

Ejercicios prácticos¶

  1. Crea un endpoint en tu aplicación Phoenix que maneje peticiones GET a la ruta /hola y responda con un mensaje de bienvenida.
  2. Crea un canal en tu aplicación Phoenix que permita a los usuarios enviar mensajes en tiempo real y mostrarlos en la pantalla.
  3. Modifica tu canal para que solo los usuarios autenticados puedan enviar mensajes.

Consejos o mejores prácticas¶

  • Utiliza endpoints para manejar diferentes tipos de peticiones en tu aplicación.
  • Utiliza canales para construir aplicaciones en tiempo real.
  • Utiliza el protocolo de websockets para establecer una conexión bidireccional y en tiempo real entre el cliente y el servidor.
  • Separa la lógica de tus canales en funciones handle_in para facilitar su mantenimiento y escalabilidad.
  • Utiliza rutas descriptivas y significativas en tus endpoints para tener un código más legible y fácil de entender.
