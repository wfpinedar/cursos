Pruebas y debugging en Go lang¶

Explicación teórica¶

Las pruebas y el debugging son dos aspectos fundamentales en el proceso de desarrollo de software. Las pruebas nos permiten verificar que nuestro código funciona correctamente y cumple con los requisitos establecidos, mientras que el debugging nos ayuda a identificar y corregir errores en nuestro código.

En Go lang, existen diversas herramientas y técnicas que nos ayudan a realizar pruebas y debugging de manera eficiente. A continuación, se presentan algunas de ellas.

Palabras clave y su definición¶

  • Pruebas unitarias: Son pruebas que se realizan de manera aislada en pequeñas partes de código, generalmente funciones o métodos, para verificar su correcto funcionamiento.
  • Integración continua: Es una práctica en la que se realizan pruebas de manera automática y constante durante el proceso de desarrollo, para detectar errores de forma temprana.
  • Depurador (Debugger): Es una herramienta que nos permite ejecutar nuestro código paso a paso, detener su ejecución en puntos específicos y examinar el estado de las variables, para identificar y corregir errores.
  • Profiling: Es una técnica que nos permite medir el rendimiento de nuestro código, identificar posibles cuellos de botella y optimizar su ejecución.

Preguntas de repaso¶

  1. ¿Qué son las pruebas unitarias y por qué son importantes en el desarrollo de software?
  2. ¿En qué consiste la integración continua y cómo puede ayudar en el proceso de desarrollo?
  3. ¿Qué es un depurador y cuál es su función en el debugging?
  4. ¿En qué consiste el profiling y cómo puede ayudar a mejorar el rendimiento de nuestro código?

Ejemplos de código en Go lang¶

Ejemplo de prueba unitaria:

func sum(a, b int) int {
    return a + b
}

func TestSum(t *testing.T) {
    result := sum(2, 3)
    if result != 5 {
        t.Errorf("Expected 5, got %d", result)
    }
}

Ejemplo de integración continua con Travis CI:

Archivo .travis.yml

language: go
go:
  - "1.15"
script:
  - go test -v ./...

Ejemplo de debugging con VS Code:

  1. Agregar un punto de interrupción en una línea de código.
  2. Ejecutar el programa en modo de depuración.
  3. Utilizar los botones de control (continuar, paso a paso, etc.) para avanzar en la ejecución del programa y examinar el estado de las variables en el panel de depuración.

Ejemplo de profiling con Go tool pprof:

func sum(a, b int) int {
    return a + b
}

func main() {
    var result int
    for i := 0; i < 100000; i++ {
        result += sum(i, i+1)
    }
}

Ejecutar en la terminal:

go test -bench=. -cpuprofile=cpu.prof
go tool pprof cpu.prof
(pprof) list sum

Ejercicios prácticos con instrucciones claras¶

  1. Crea una función en Go lang que reciba dos números enteros y devuelva la suma de ambos.
  2. Realiza una prueba unitaria para verificar que la función anterior devuelve el resultado esperado.
  3. Configura un proyecto en Travis CI para que realice las pruebas de manera automática cada vez que se haga un push al repositorio.
  4. Agrega un punto de interrupción en una línea de código en un proyecto de Go lang y utiliza el depurador para examinar el estado de una variable en ese punto.
  5. Utiliza la herramienta de profiling para medir el rendimiento de una función en tu código y realiza las optimizaciones necesarias para mejorar su ejecución.

Consejos o mejores prácticas¶

  • Realizar pruebas unitarias en todas las funciones y métodos de nuestro código.
  • Utilizar herramientas de integración continua para automatizar las pruebas y detectar errores de manera temprana.
  • Aprender a utilizar un depurador para identificar y corregir errores de forma eficiente.
  • Utilizar técnicas de profiling para mejorar el rendimiento de nuestro código.
  • Realizar pruebas de manera constante y mantener un buen nivel de cobertura de pruebas en nuestro código.
