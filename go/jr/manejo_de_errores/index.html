Manejo de errores en Go lang¶

El manejo de errores es una parte fundamental en cualquier lenguaje de programación, ya que permite controlar y manejar situaciones inesperadas que pueden ocurrir durante la ejecución de un programa. En Go lang, el manejo de errores se realiza a través de estructuras de errores y manejo de excepciones.

Explicación teórica¶

En Go lang, los errores se manejan a través de la interfaz error, que es una estructura que contiene un mensaje descriptivo del error. Esta interfaz se puede utilizar para crear tipos de errores personalizados o para devolver errores específicos en funciones y métodos.

Para manejar los errores, se utiliza la declaración if para verificar si el valor de error es diferente de nil, lo que indica que ha ocurrido un error. En caso de que haya ocurrido un error, se puede utilizar la declaración log.Fatal() para imprimir el mensaje de error y terminar la ejecución del programa.

Además, Go lang también cuenta con la declaración defer que permite ejecutar una función al final de una función, incluso si ocurre un error durante su ejecución. Esto es útil para realizar tareas de limpieza o cierre de recursos.

Palabras clave y su definición¶

  • Error: Una situación inesperada o no deseada que ocurre durante la ejecución de un programa.
  • Interfaz error: Una estructura que contiene un mensaje descriptivo del error.
  • nil: Un valor nulo que indica que no ha ocurrido ningún error.
  • Declaración if: Una estructura de control que permite evaluar una condición y ejecutar un bloque de código si la condición es verdadera.
  • Declaración log.Fatal(): Una función que imprime un mensaje de error y termina la ejecución del programa.
  • Declaración defer: Una estructura de control que permite ejecutar una función al final de una función, incluso si ocurre un error durante su ejecución.

Ejemplos de código en Go lang¶

  1. Creación de un error personalizado:
package main

import (
    "errors"
    "fmt"
)

func divide(dividend, divisor float64) (float64, error) {
    if divisor == 0 {
        return 0, errors.New("No se puede dividir por cero")
    }
    return dividend / divisor, nil
}

func main() {
    result, err := divide(10, 0)
    if err != nil {
        fmt.Println(err)
    } else {
        fmt.Println(result)
    }
}
  1. Manejo de errores con la declaración if y log.Fatal():
package main

import (
    "log"
)

func divide(dividend, divisor float64) float64 {
    if divisor == 0 {
        log.Fatal("No se puede dividir por cero")
    }
    return dividend / divisor
}

func main() {
    result := divide(10, 0)
}

Consejos o mejores prácticas¶

  • Utiliza errores personalizados para proporcionar información más específica sobre el error que ha ocurrido.
  • Utiliza la declaración defer para realizar tareas de limpieza o cierre de recursos.
  • Evita manejar los errores de manera genérica, trata de manejarlos de manera específica según el tipo de error que pueda ocurrir.
