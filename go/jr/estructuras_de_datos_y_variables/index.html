Estructuras de datos y variables en Go¶

Descripción del módulo¶

En este módulo, aprenderás sobre los tipos de datos, variables y constantes en Go. Además, te familiarizarás con el uso de estructuras de datos básicas como arrays, slices y maps. Esta lección es esencial para comprender cómo almacenar y manipular datos en Go y es fundamental para cualquier programador en este lenguaje.

Explicación teórica¶

Tipos de datos¶

Los tipos de datos en Go se dividen en dos categorías: tipos básicos y tipos compuestos. Los tipos básicos son los tipos de datos más simples y se utilizan para representar valores numéricos, booleanos y cadenas de texto. Algunos ejemplos de tipos básicos en Go son int para números enteros, float para números de punto flotante y string para cadenas de texto.

Por otro lado, los tipos compuestos son tipos de datos que están formados por la combinación de uno o más tipos básicos. Algunos ejemplos de tipos compuestos en Go son arrays, slices y maps, que se explicarán más adelante en este módulo.

Variables y constantes¶

En Go, una variable es un nombre que se utiliza para almacenar un valor en la memoria. Para declarar una variable en Go, se utiliza la siguiente sintaxis:

var nombre_variable tipo_dato

Por ejemplo, para declarar una variable llamada edad de tipo int, se utilizaría la siguiente línea de código:

var edad int

Además de las variables, también se pueden utilizar constantes en Go. Una constante es similar a una variable en el sentido de que también se utiliza para almacenar un valor en la memoria, pero a diferencia de una variable, su valor no puede cambiar durante la ejecución del programa. Para declarar una constante en Go, se utiliza la siguiente sintaxis:

const nombre_constante tipo_dato = valor

Por ejemplo, para declarar una constante llamada pi de tipo float con valor 3.14, se utilizaría la siguiente línea de código:

const pi float = 3.14

Estructuras de datos básicas¶

Arrays¶

Un array en Go es una estructura de datos que se utiliza para almacenar una colección de elementos del mismo tipo. La principal característica de los arrays es que tienen una longitud fija, lo que significa que una vez definidos, no se pueden modificar. La sintaxis para declarar un array en Go es la siguiente:

var nombre_array [longitud]tipo_dato

Donde nombre_array es el nombre del array, longitud es el número de elementos que se pueden almacenar en el array y tipo_dato es el tipo de dato de los elementos que se almacenarán en el array.

Por ejemplo, para declarar un array llamado numeros con una longitud de 5 y de tipo int, se utilizaría la siguiente línea de código:

var numeros [5]int

Slices¶

Un slice en Go es una estructura de datos similar a un array, pero con una longitud variable. Esto significa que se pueden agregar o eliminar elementos del slice durante la ejecución del programa. La sintaxis para declarar un slice en Go es la siguiente:

var nombre_slice []tipo_dato

Donde nombre_slice es el nombre del slice y tipo_dato es el tipo de dato de los elementos que se almacenarán en el slice.

Por ejemplo, para declarar un slice llamado nombres de tipo string, se utilizaría la siguiente línea de código:

var nombres []string

Maps¶

Un map en Go es una estructura de datos que se utiliza para almacenar una colección de pares clave-valor. Esto significa que cada elemento en un map tiene una clave única que se utiliza para acceder a su valor correspondiente. La sintaxis para declarar un map en Go es la siguiente:

var nombre_map map[tipo_clave]tipo_valor

Donde nombre_map es el nombre del map, tipo_clave es el tipo de dato de las claves y tipo_valor es el tipo de dato de los valores.

Por ejemplo, para declarar un map llamado edades con claves de tipo string y valores de tipo int, se utilizaría la siguiente línea de código:

var edades map[string]int

Palabras clave y su definición¶

  • int: tipo de dato que representa números enteros en Go.
  • float: tipo de dato que representa números de punto flotante en Go.
  • string: tipo de dato que representa cadenas de texto en Go.
  • var: palabra clave utilizada para declarar variables en Go.
  • const: palabra clave utilizada para declarar constantes en Go.
  • array: estructura de datos que almacena una colección de elementos del mismo tipo con una longitud fija.
  • slice: estructura de datos que almacena una colección de elementos del mismo tipo con una longitud variable.
  • map: estructura de datos que almacena una colección de pares clave-valor.

Ejemplos de código en Go lang¶

Declaración de una variable y una constante¶

var nombre string // declaración de una variable de tipo string
nombre = "Juan" // asignación del valor "Juan" a la variable nombre

const pi float = 3.14 // declaración de una constante de tipo float con valor 3.14

Declaración de un array y un slice¶

var numeros [5]int // declaración de un array de tipo int con longitud 5
numeros[0] = 1 // asignación del valor 1 al primer elemento del array
numeros[1] = 2 // asignación del valor 2 al segundo elemento del array

var nombres []string // declaración de un slice de tipo string
nombres = append(nombres, "Ana") // agregamos el valor "Ana" al slice
nombres = append(nombres, "Carlos") // agregamos el valor "Carlos" al slice

Declaración de un map¶

var edades map[string]int // declaración de un map con claves de tipo string y valores de tipo int
edades["Ana"] = 25 // asignación del valor 25 a la clave "Ana"
edades["Carlos"] = 30 // asignación del valor 30 a la clave "Carlos"

Ejercicios prácticos¶

  1. Crea un programa en Go que declare e imprima una variable de cada tipo básico.
  2. Crea una constante en Go con el valor de tu año de nacimiento y utiliza una variable para calcular tu edad y mostrarla en pantalla.
  3. Crea un programa en Go que declare un array de 5 números enteros y los muestre en pantalla.
  4. Crea un programa en Go que declare un slice de nombres y le permita al usuario agregar nuevos nombres a través de la entrada estándar.
  5. Crea un programa en Go que declare un map con tus comidas favoritas y sus precios, y luego muestre la información en pantalla.

Consejos o mejores prácticas¶

  • Utiliza nombres descriptivos para tus variables y constantes para que sea más fácil entender tu código.
  • Aprovecha el poder de los slices y maps en lugar de utilizar arrays, ya que te permiten tener una mayor flexibilidad en la manipulación de datos.
  • Utiliza constantes en lugar de valores numéricos o cadenas de texto directamente en tu código, ya que esto ayuda a que tu código sea más legible y fácil de mantener.
  • No olvides inicializar tus arrays, slices y maps antes de utilizarlos para evitar errores de ejecución.
