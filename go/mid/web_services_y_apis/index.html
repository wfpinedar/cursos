Web services y APIs¶

Descripción¶

En el mundo actual de la tecnología, la comunicación entre sistemas es esencial para el funcionamiento de aplicaciones y servicios. Los web services y las APIs (Application Programming Interfaces) son herramientas que permiten la comunicación entre aplicaciones de manera eficiente y segura. En este módulo, aprenderás cómo crear y consumir servicios web y APIs utilizando el lenguaje de programación Go.

Teoría¶

Web services¶

Un web service es un conjunto de tecnologías y estándares que permiten la comunicación entre sistemas a través de una red, como Internet. Estos servicios se basan en el protocolo HTTP y utilizan formatos de intercambio de datos como XML o JSON para transmitir información. Los web services pueden ser usados para compartir datos o funcionalidades entre aplicaciones, sin importar el lenguaje de programación utilizado.

Existen diferentes tipos de web services, entre los que destacan:

  • REST (Representational State Transfer): es un estilo de arquitectura de software que define cómo se deben comunicar los sistemas a través de la web. Utiliza los métodos HTTP (GET, POST, PUT, DELETE) para realizar operaciones sobre los recursos.
  • SOAP (Simple Object Access Protocol): es un protocolo que define cómo se debe estructurar la información para ser intercambiada entre sistemas. Utiliza XML para el intercambio de datos y requiere un formato de mensaje específico.
  • gRPC: es un framework de código abierto desarrollado por Google que permite la comunicación entre sistemas de manera eficiente y segura utilizando RPC (Remote Procedure Call).

APIs¶

Las APIs son un conjunto de reglas y protocolos que definen cómo se debe interactuar con una aplicación o servicio. Estas interfaces permiten a los desarrolladores acceder a ciertas funciones o datos de una aplicación de manera controlada y segura. Las APIs pueden ser públicas o privadas, y pueden ser utilizadas para compartir datos o funcionalidades entre aplicaciones.

Palabras clave y definiciones¶

  • Web service: conjunto de tecnologías y estándares que permiten la comunicación entre sistemas a través de una red.
  • API: conjunto de reglas y protocolos que definen cómo se debe interactuar con una aplicación o servicio.
  • REST: estilo de arquitectura de software que define cómo se deben comunicar los sistemas a través de la web.
  • SOAP: protocolo que define cómo se debe estructurar la información para ser intercambiada entre sistemas.
  • gRPC: framework de código abierto que permite la comunicación entre sistemas utilizando RPC.
  • RPC: Remote Procedure Call, método de comunicación entre procesos que permite a un programa llamar a una función en otro proceso.
  • HTTP: Hypertext Transfer Protocol, protocolo de comunicación utilizado en la web.
  • XML: Extensible Markup Language, lenguaje de marcado utilizado para estructurar datos.
  • JSON: JavaScript Object Notation, formato de intercambio de datos ligero y fácil de leer.
  • API pública: interfaz que permite a cualquier desarrollador acceder a sus funciones o datos.
  • API privada: interfaz que requiere autorización para ser utilizada y solo está disponible para ciertos desarrolladores o aplicaciones.

Ejemplos de código¶

Creación de un servidor REST utilizando el framework Gin¶

package main

import (
    "github.com/gin-gonic/gin"
)

func main() {
    // Creamos un router utilizando el framework Gin
    router := gin.Default()

    // Definimos una ruta GET con el endpoint "/saludo"
    router.GET("/saludo", func(c *gin.Context) {
        // Enviamos un mensaje de saludo como respuesta
        c.JSON(200, gin.H{
            "mensaje": "¡Hola desde mi servidor REST!",
        })
    })

    // Iniciamos el servidor en el puerto 8080
    router.Run(":8080")
}

Consumo de una API utilizando la librería net/http¶

package main

import (
    "encoding/json"
    "fmt"
    "net/http"
)

func main() {
    // Realizamos una petición GET a la API
    response, err := http.Get("https://jsonplaceholder.typicode.com/posts/1")

    if err != nil {
        fmt.Println(err)
    }

    // Creamos una estructura para almacenar la respuesta
    var post struct {
        UserID int    `json:"userId"`
        ID     int    `json:"id"`
        Title  string `json:"title"`
        Body   string `json:"body"`
    }

    // Decodificamos la respuesta en formato JSON y la almacenamos en la estructura
    err = json.NewDecoder(response.Body).Decode(&post)

    if err != nil {
        fmt.Println(err)
    }

    // Imprimimos el título y el cuerpo del post
    fmt.Println(post.Title)
    fmt.Println(post.Body)
}

Creación de un servidor gRPC¶

package main

import (
    "context"
    "fmt"
    "log"
    "net"

    "google.golang.org/grpc"

    "example.com/server/proto"
)

// Implementamos la interfaz generada por gRPC
type server struct{}

// Definimos un método para saludar al cliente
func (s *server) Saludar(ctx context.Context, req *proto.SaludoRequest) (*proto.SaludoResponse, error) {
    // Obtenemos el nombre del cliente desde la petición
    nombre := req.GetNombre()
    // Creamos un mensaje de saludo y lo enviamos como respuesta
    mensaje := fmt.Sprintf("¡Hola %s desde mi servidor gRPC!", nombre)
    return &proto.SaludoResponse{Mensaje: mensaje}, nil
}

func main() {
    // Creamos un servidor gRPC y lo registramos en el puerto 50051
    servidor := grpc.NewServer()
    proto.RegisterSaludoServiceServer(servidor, &server{})
    direccion, err := net.Listen("tcp", ":50051")

    if err != nil {
        log.Fatalf("Error al iniciar el servidor: %v", err)
    }

    // Iniciamos el servidor
    if err := servidor.Serve(direccion); err != nil {
        log.Fatalf("Error al iniciar el servidor: %v", err)
    }
}

Consumo de un servicio gRPC¶

package main

import (
    "context"
    "fmt"
    "log"

    "google.golang.org/grpc"

    "example.com/client/proto"
)

func main() {
    // Creamos una conexión con el servidor gRPC
    conn, err := grpc.Dial(":50051", grpc.WithInsecure())

    if err != nil {
        log.Fatalf("Error al conectar con el servidor: %v", err)
    }
    defer conn.Close()
    // Creamos un cliente utilizando la conexión
    cliente := proto.NewSaludoServiceClient(conn)

    // Realizamos una petición al servidor
    respuesta, err := cliente.Saludar(context.Background(), &proto.SaludoRequest{Nombre: "Juan"})

    if err != nil {
        log.Fatalf("Error al llamar al servidor: %v", err)
    }

    // Imprimimos el mensaje de respuesta
    fmt.Println(respuesta.GetMensaje())
}

Ejercicios prácticos¶

  1. Crea un servidor REST utilizando el framework Gin que permita realizar operaciones CRUD (Create, Read, Update, Delete) sobre una lista de tareas.
  2. Consuma la API pública de OpenWeatherMap para obtener la información del clima de una ciudad específica y la muestre por consola.
  3. Crea un servidor gRPC que permita realizar la operación de suma de dos números y devuelve el resultado como respuesta.

Consejos y mejores prácticas¶

  • Utiliza el framework o librería que mejor se adapte a tus necesidades, ya sea para crear o consumir servicios web y APIs.
  • Documenta tus servicios y APIs de manera clara y concisa, incluyendo la estructura de los datos que se envían y reciben.
  • Valida siempre los datos que se envían y reciben para evitar errores y posibles vulnerabilidades.
  • Utiliza HTTPS para proteger la comunicación entre sistemas y asegurarte de que los datos no sean interceptados por terceros.
  • Realiza pruebas exhaustivas para garantizar el correcto funcionamiento de tus servicios y APIs antes de ponerlos en producción.
  • Sigue las mejores prácticas y estándares establecidos para asegurar la interoperabilidad y la compatibilidad con otros sistemas.
