Programación funcional en Go¶

La programación funcional es un paradigma de programación que se basa en el uso de funciones como elementos fundamentales del desarrollo de software. A diferencia de otros paradigmas, en la programación funcional se enfoca en la evaluación de expresiones y en la ausencia de cambio de estado y datos mutables. En Go, un lenguaje de programación compilado y concurrente, también es posible utilizar conceptos de programación funcional para escribir código más conciso y legible.

Palabras clave y definiciones¶

  • Función de orden superior: Una función que puede recibir como parámetros otras funciones o devolver una función como resultado. En Go, se pueden utilizar funciones de orden superior para crear abstracciones y reutilizar código.
  • Closures: Una función anónima que puede acceder y modificar variables definidas fuera de su alcance. En Go, los closures se utilizan comúnmente para crear funciones que recuerden el estado de una variable.
  • Evaluación de expresiones: El proceso de calcular el valor de una expresión matemática o lógica. En la programación funcional, se prioriza la evaluación de expresiones sobre la ejecución de instrucciones.
  • Estado y datos mutables: La capacidad de cambiar el valor de una variable o estructura de datos durante la ejecución del programa. En la programación funcional, se evita el uso de datos mutables para prevenir efectos secundarios y aumentar la seguridad del código.

Ejemplos de código en Go¶

Funciones de orden superior¶

En el siguiente ejemplo, se define una función map que recibe como parámetros una función f y un slice s de enteros, y devuelve un nuevo slice con los resultados de aplicar la función f a cada elemento de s.

func map(f func(int) int, s []int) []int {
    result := make([]int, len(s))
    for i, v := range s {
        result[i] = f(v)
    }
    return result
}

func main() {
    numbers := []int{1, 2, 3, 4, 5}
    // Se define una función anónima que multiplica por 2 cada elemento.
    doubled := map(func(x int) int {
        return x * 2
    }, numbers)
    fmt.Println(doubled) // Output: [2 4 6 8 10]
}

Closures¶

En este ejemplo, se declara una función counter que devuelve una función anónima que incrementa un contador cada vez que es llamada.

func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

func main() {
    nextNum := counter()
    fmt.Println(nextNum()) // Output: 1
    fmt.Println(nextNum()) // Output: 2
    fmt.Println(nextNum()) // Output: 3
}

Ejercicios prácticos¶

  1. Crea una función de orden superior llamada filter que reciba como parámetros una función f y un slice de strings s, y devuelva un nuevo slice con los elementos de s que cumplan con la condición de f.
  2. Utiliza closures para crear una función calculator que reciba un operando y devuelva una función que realice una operación matemática con ese operando.
  3. Escribe una función de orden superior llamada reduce que reciba como parámetros una función f, un valor inicial y un slice de enteros s, y devuelva el resultado de aplicar la función f a cada elemento de s y el valor inicial.
  4. Crea una función makeSuffixer que reciba una cadena de texto suffix y devuelva una función que añada el sufijo suffix a cualquier cadena de texto que se le pase como argumento.

Consejos y mejores prácticas¶

  • Utiliza funciones de orden superior y closures para crear abstracciones y evitar la repetición de código.
  • Evita el uso de variables mutables para mantener un código más seguro y prevenir efectos secundarios.
  • Practica la evaluación de expresiones y el uso de funciones como elementos fundamentales de la programación funcional en Go.
