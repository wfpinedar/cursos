Proyecto final¶

En este módulo se llevará a cabo un proyecto final en el que se pondrán en práctica todos los conocimientos adquiridos sobre Luminus Framework. El objetivo es desarrollar una aplicación completa utilizando esta herramienta y presentarla al final del curso.

Explicación teórica¶

Luminus Framework es un framework de desarrollo web basado en Clojure que permite crear aplicaciones de forma rápida y eficiente. Está construido sobre otros frameworks y bibliotecas como Ring, Compojure y Hiccup, lo que lo convierte en una opción poderosa para el desarrollo de aplicaciones web robustas.

El enfoque de Luminus se basa en la simplicidad, la modularidad y el rendimiento. Esto se logra gracias a su arquitectura basada en componentes, que permite la reutilización de código y la fácil integración de nuevas funcionalidades.

Para llevar a cabo el proyecto final, es importante tener en cuenta los siguientes conceptos clave:

Palabras clave y su definición¶

  • Clojure: es un lenguaje de programación funcional dinámico que se ejecuta en la máquina virtual de Java (JVM). Es el lenguaje en el que está basado Luminus Framework.

  • Ring: es una biblioteca de manejo de peticiones HTTP para aplicaciones web en Clojure. Proporciona una interfaz simple y flexible para manejar las solicitudes y las respuestas.

  • Compojure: es un framework de enrutamiento para aplicaciones web en Clojure. Facilita la definición de rutas y manejo de peticiones HTTP.

  • Hiccup: es una biblioteca de generación de HTML en Clojure, que permite definir el contenido de una página web como una estructura de datos.

  • Componentes: es el enfoque de arquitectura utilizado por Luminus, donde cada funcionalidad se divide en pequeñas piezas independientes, lo que facilita su mantenimiento y reutilización.

  • Leiningen: es un sistema de construcción y administración de dependencias para proyectos en Clojure. Se utiliza para crear y ejecutar aplicaciones desarrolladas con Luminus.

Ejemplos de código en Luminus Framework¶

Para comprender mejor cómo se utiliza Luminus en el desarrollo de aplicaciones web, a continuación se presentan algunos ejemplos de código:

Creación de un proyecto con Leiningen¶

Para crear un proyecto en Luminus utilizando Leiningen, se debe ejecutar el siguiente comando en la terminal:

lein new luminus <nombre-proyecto> +api +h2 +reagent +sqlite +swagger

Esto creará un proyecto con las características especificadas (API, base de datos H2, librería Reagent para manejo de JavaScript y base de datos SQLite) y también incluirá la documentación de Swagger para la API.

Definición de rutas con Compojure¶

Para definir una ruta en una aplicación web con Luminus y Compojure, se debe utilizar la macro defroutes de la siguiente manera:

(defroutes app-routes
  (GET "/hello" [] "Hola mundo")
  (GET "/user/:id" [id] (str "El ID del usuario es " id))
  (POST "/user" {body :params} (str "El nombre del usuario es " (:name body))))

En este ejemplo, se definen tres rutas: una para obtener un saludo, otra para obtener el ID de un usuario y otra para crear un usuario con un nombre especificado en el cuerpo de la petición.

Uso de Hiccup para generar HTML¶

Hiccup permite crear HTML utilizando una sintaxis similar a Clojure. Por ejemplo, para crear una lista de elementos, se puede utilizar la siguiente estructura de datos:

[:ul
  [:li "Elemento 1"]
  [:li "Elemento 2"]
  [:li "Elemento 3"]]

Esto generará el siguiente HTML:

<ul>
  <li>Elemento 1</li>
  <li>Elemento 2</li>
  <li>Elemento 3</li>
</ul>

Ejercicios prácticos con instrucciones claras¶

  1. Crear un proyecto en Luminus utilizando Leiningen y las características de su preferencia.
  2. Definir al menos tres rutas en el proyecto utilizando Compojure.
  3. Utilizar Hiccup para generar una página de inicio con contenido HTML.
  4. Agregar una funcionalidad de base de datos utilizando la base de datos de su elección (por ejemplo, PostgreSQL o MySQL).
  5. Implementar una API utilizando Swagger y probarla con una herramienta como Postman.

Consejos o mejores prácticas¶

  • Es recomendable seguir la convención de nombres de Luminus, que utiliza el formato nombre-proyecto para el nombre del proyecto y nombre-paquete para el nombre del paquete.
  • Utilizar componentes para dividir la funcionalidad en pequeñas piezas y facilitar su reutilización.
  • Utilizar las herramientas incluidas en el proyecto (como Swagger) para documentar y probar la API.
  • Seguir las buenas prácticas de Clojure, como utilizar funciones puras y evitar mutar el estado.
  • Utilizar la documentación oficial de Luminus y su comunidad en línea para resolver dudas y obtener ayuda en caso necesario.

En resumen, el proyecto final en Luminus Framework brindará la oportunidad de aplicar todos los conocimientos adquiridos durante el curso y crear una aplicación web completa utilizando esta herramienta poderosa. Se recomienda seguir las buenas prácticas y utilizar las herramientas adecuadas para obtener un desarrollo eficiente y de calidad.
