Creación de controladores en Luminus Framework¶

Descripción teórica¶

Los controladores son una parte fundamental en el desarrollo de aplicaciones web, ya que se encargan de gestionar las solicitudes de los usuarios y devolverles una respuesta adecuada. En Luminus Framework, los controladores se basan en el patrón MVC (Modelo-Vista-Controlador), donde el controlador actúa como intermediario entre la vista y el modelo.

En pocas palabras, el controlador es el encargado de recibir las solicitudes de los usuarios, procesarlas y enviar una respuesta al navegador. Esta respuesta puede ser una página HTML, un archivo estático, un mensaje de error, entre otros.

Palabras clave y su definición¶

  • Controlador: Es una clase que gestiona las solicitudes de los usuarios y devuelve una respuesta.
  • Rutas: Son las direcciones URL que se utilizan para acceder a los diferentes recursos de la aplicación.
  • Métodos HTTP: Son las acciones que se pueden realizar sobre una ruta, como GET, POST, PUT, DELETE, entre otros.
  • Parámetros: Son valores que se envían junto con la solicitud y que se utilizan para realizar acciones específicas en el controlador.
  • Middleware: Son funciones que se ejecutan antes o después de una solicitud, y permiten realizar tareas como autenticación, manejo de errores, entre otros.

Ejemplos de código en Luminus Framework¶

A continuación, se muestra un ejemplo de un controlador en Luminus que maneja la solicitud de la ruta "/users" y devuelve una lista de usuarios en formato JSON:

(ns my-app.controllers.users
  (:require [ring.util.response :as response]))

(defn get-users [request]
  (let [users (db/get-users)] ; Se obtienen los usuarios desde la base de datos
    (response/json-response users))) ; Se devuelve una respuesta en formato JSON

En este ejemplo, se utiliza el método GET para obtener la lista de usuarios, y se utiliza el middleware response/json-response para devolver una respuesta en formato JSON.

Ejercicios prácticos con instrucciones claras¶

  1. Crea un nuevo proyecto en Luminus y configura una ruta "/hello" que devuelva una página HTML con el mensaje "Hola mundo".
  2. Crea un controlador que maneje la solicitud de la ruta "/users/:id" y devuelva los datos del usuario con el ID especificado en formato JSON.
  3. Agrega un middleware que verifique si el usuario tiene permisos para acceder a ciertas rutas de la aplicación.
  4. Crea un formulario de registro de usuarios que envíe los datos al controlador correspondiente para ser almacenados en la base de datos.

Consejos o mejores prácticas¶

  • Utiliza nombres descriptivos para tus controladores y métodos, por ejemplo: users-controller y get-users.
  • Divide tus controladores en funciones más pequeñas para facilitar su mantenimiento.
  • Utiliza middlewares para realizar tareas comunes como autenticación, manejo de errores, entre otros.
  • Utiliza la función defroutes para definir todas las rutas de tu aplicación en un solo lugar.
  • Aprovecha al máximo las funcionalidades de Luminus, como la gestión de dependencias y la integración con otras bibliotecas.
