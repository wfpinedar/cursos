Administración de base de datos en Luminus Framework¶

Descripción del módulo¶

En este módulo se explicará cómo trabajar con la base de datos en Luminus Framework. Se verá cómo configurar y conectarse a una base de datos, cómo crear modelos y cómo realizar consultas utilizando SQL o un ORM.

Explicación teórica¶

Luminus Framework utiliza una capa de persistencia basada en el framework de Java, Hibernate. Esto significa que puedes trabajar con bases de datos relacionales utilizando SQL o utilizar un ORM (Object-Relational Mapping) para interactuar con la base de datos de una manera más orientada a objetos. En Luminus, puedes utilizar cualquier base de datos relacional compatible con JDBC, como PostgreSQL, MySQL, Oracle, entre otros.

La administración de la base de datos en Luminus se divide en tres partes principales: configuración y conexión, creación de modelos y consultas.

Configuración y conexión¶

Para configurar y conectarse a una base de datos en Luminus, primero debes especificar la dependencia de la base de datos en tu proyecto utilizando Leiningen o Maven. Luego, debes configurar la conexión en el archivo "project.clj" o "pom.xml" según el gestor de dependencias que estés utilizando. Una vez configurada la conexión, puedes crear una instancia de la base de datos en tu aplicación utilizando el archivo "resources/config.edn". Luminus también proporciona una función llamada "migrate" que te permite crear automáticamente las tablas y relaciones en la base de datos basándose en tus modelos.

Creación de modelos¶

En Luminus, los modelos son clases que representan una tabla en la base de datos. Puedes crearlos manualmente utilizando anotaciones de Hibernate o mediante la generación automática de código utilizando un plugin de Luminus. Los modelos también pueden tener relaciones entre sí, lo que facilita la creación de consultas más complejas.

Consultas¶

Para realizar consultas a la base de datos en Luminus, puedes usar SQL directamente o utilizar un ORM como Hibernate. El uso de un ORM te permite escribir consultas de manera más orientada a objetos, lo que puede facilitar la lectura y el mantenimiento del código. Luminus también proporciona una biblioteca llamada "Korma" que te permite escribir consultas en un estilo más declarativo.

Palabras clave y su definición¶

  • Base de datos: un sistema de almacenamiento de datos estructurado que permite la creación, lectura, actualización y eliminación de información de manera eficiente.
  • Hibernate: un framework de mapeo objeto-relacional para Java que permite interactuar con una base de datos relacional utilizando objetos.
  • JDBC: Java Database Connectivity, una API de Java que permite la conexión y ejecución de consultas a bases de datos relacionales.
  • ORM (Object-Relational Mapping): una técnica de programación que permite mapear objetos en una base de datos relacional y viceversa.
  • Leiningen y Maven: gestores de dependencias para proyectos en Clojure y Java, respectivamente.
  • Plugin: una extensión de software que agrega funcionalidad a una aplicación existente.
  • Anotaciones: etiquetas utilizadas en el código para agregar metadatos o comportamiento adicional.
  • Korma: una biblioteca de Luminus que permite escribir consultas en un estilo más declarativo.

Ejemplos de código en Luminus Framework¶

Configuración y conexión a una base de datos¶

;; En project.clj
:dependencies [[org.clojure/java.jdbc "0.7.11"]
               [org.clojure/java.jdbc-mysql "0.7.11"]]

;; En config.edn
{:database-url "jdbc:mysql://localhost:3306/mydb"
 :database-options {:user "username" :password "password"}}

;; Crear instancia de la base de datos
(def db (luminus.db.core/db))

;; Ejecutar migraciones
(luminus.db.migration/migrate db)

Creación de modelos¶

;; Definir modelo con anotaciones de Hibernate
(ns myapp.models.user
  (:require [luminus.db.core :as db]
            [org.hibernate.annotations :refer :all]))

(defrecord User []
  Object
  (toString [_] (str "User: " (:username this))))

(defn user []
  (db/entity User
    (id :id)
    (property :username String)
    (property :password String)))

;; Generar modelo con plugin de Luminus
(lein generate model user)

Consultas con Korma¶

;; Consulta utilizando Korma
(require '[korma.db :refer [exec-raw]]
         '[korma.core :refer :all])

(defn get-users []
  (select users))

(defn get-users-by-id [id]
  (select users
    (where {:id id})))

Ejercicios prácticos¶

  1. Configura y conecta tu aplicación de Luminus a una base de datos PostgreSQL.
  2. Crea un modelo para una tabla "producto" con los campos "nombre", "precio" y "cantidad".
  3. Escribe una consulta utilizando Korma para obtener todos los productos con un precio mayor a $50.
  4. Agrega una nueva columna llamada "descripcion" a la tabla "producto" y actualiza tu modelo y consultas para reflejar este cambio.

Consejos o mejores prácticas¶

  • Utiliza modelos y relaciones entre ellos para facilitar la creación de consultas complejas.
  • Si estás utilizando un ORM, asegúrate de comprender cómo se traducen tus consultas a SQL para evitar posibles problemas de rendimiento.
  • Utiliza la función "migrate" para crear automáticamente tablas y relaciones en la base de datos basándose en tus modelos.
  • Utiliza la biblioteca Korma para escribir consultas de manera más declarativa y legible.
