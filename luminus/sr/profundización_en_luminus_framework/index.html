Profundización en Luminus Framework¶

En este módulo nos adentraremos en los conceptos avanzados de Luminus Framework, un framework de desarrollo web escrito en el lenguaje de programación Clojure. Aprenderemos sobre su funcionamiento interno y cómo extender sus capacidades mediante la creación de plugins.

Explicación teórica¶

Luminus Framework se basa en el patrón de arquitectura de software "Modelo Vista Controlador" (MVC). Esto significa que divide la aplicación en tres componentes principales: el modelo, la vista y el controlador. El modelo se encarga de la lógica de la aplicación y la manipulación de los datos, la vista se encarga de la presentación de la información y el controlador actúa como intermediario entre el modelo y la vista.

Una de las características principales de Luminus es su énfasis en la simplicidad y en seguir el principio de "convención sobre configuración". Esto significa que el desarrollador no tiene que preocuparse por configurar la aplicación en exceso, ya que el framework asume ciertas convenciones de configuración por defecto.

Luminus utiliza el servidor web Jetty y el sistema de plantillas Hiccup, lo que lo hace muy ligero y rápido. Además, cuenta con una amplia documentación y una comunidad activa que lo respalda.

Palabras clave y su definición¶

  • MVC: Modelo Vista Controlador, un patrón de arquitectura de software en el que la aplicación se divide en tres componentes principales: modelo, vista y controlador.
  • Convención sobre configuración: principio en el que se asume que existen ciertas convenciones de configuración por defecto, lo que facilita el desarrollo y reduce la cantidad de código necesario.
  • Jetty: servidor web utilizado por Luminus para manejar las solicitudes HTTP.
  • Hiccup: sistema de plantillas utilizado por Luminus para generar contenido HTML.

Ejemplos de código en Luminus Framework¶

A continuación se muestra un ejemplo de un controlador en Luminus que maneja una solicitud GET y devuelve una vista con algunos datos:

(ns my-app.controllers.hello)

(defn hello-view [name]
  [:h1 "¡Hola, " name "!"])

(defn hello-world [request]
  (let [name (get-in request [:params :name])
        view (hello-view name)]
    {:status 200
     :headers {"Content-Type" "text/html"}
     :body (hiccup.core/html view)}))

Ejercicios prácticos con instrucciones claras¶

  1. Crear un nuevo proyecto en Luminus utilizando el comando lein new luminus my-app. Este comando creará una estructura de proyecto básica y descargará las dependencias necesarias.
  2. En el archivo project.clj, agregar la dependencia para utilizar la base de datos MySQL: [mysql/mysql-connector-java "5.1.38"].
  3. En el archivo src/my_app/db.clj, configurar la conexión a la base de datos y escribir una función que devuelva una lista con todos los usuarios de la tabla users.
  4. Crear un controlador que maneje una solicitud GET a la ruta "/users" y llame a la función creada en el paso anterior para obtener la lista de usuarios.
  5. Crear una vista que muestre la lista de usuarios en una tabla HTML.
  6. Ejecutar el proyecto y comprobar que al acceder a la ruta "/users" se muestra la lista de usuarios correctamente.

Consejos o mejores prácticas¶

  • Utilizar las convenciones de configuración por defecto de Luminus siempre que sea posible, ya que esto reduce la cantidad de código necesario y facilita el desarrollo.
  • Revisar la documentación oficial y la comunidad en línea para encontrar soluciones a posibles problemas o dudas.
  • Utilizar la estructura de proyecto recomendada por Luminus para mantener un código organizado y fácil de mantener.
