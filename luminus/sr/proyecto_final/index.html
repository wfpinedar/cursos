Proyecto final¶

Descripción del módulo¶

En este módulo final del curso de Luminus Framework, se pondrán en práctica todos los conceptos aprendidos a lo largo del curso para desarrollar un proyecto completo utilizando esta herramienta. Este proyecto final servirá como una evaluación y una oportunidad para aplicar los conocimientos adquiridos en un escenario real.

Explicación teórica¶

El proyecto final consistirá en la creación de una aplicación web utilizando Luminus Framework. Esta aplicación tendrá como objetivo gestionar una lista de tareas, permitiendo a los usuarios crear, editar y eliminar tareas, así como también marcarlas como completadas.

Para desarrollar este proyecto, se utilizarán los conceptos aprendidos en los módulos anteriores, como la estructura de un proyecto en Luminus, la creación de rutas, la utilización de plantillas HTML, el acceso a bases de datos, entre otros.

Palabras clave y su definición¶

  • Luminus Framework: Es un framework de aplicaciones web para el lenguaje de programación Clojure.
  • Proyecto final: Es una actividad en la que se aplican todos los conocimientos adquiridos en un curso o módulo.
  • Aplicación web: Es un software accesible a través de un navegador web que permite realizar diferentes tareas o funciones.
  • Gestión de tareas: Es el proceso de organizar y administrar las tareas de una manera eficiente y efectiva.

Ejemplos de código en Luminus Framework¶

  1. Creación de una ruta en Luminus:
(defroutes app-routes
  (GET "/" [] (home-page))
  (POST "/task" [task] (create-task task))
  (PUT "/task/:id" [id task] (update-task id task))
  (DELETE "/task/:id" [id] (delete-task id)))
  1. Utilización de plantillas HTML en Luminus:
(defn home-page []
  (layout/common
    [:h1 "Lista de tareas"]
    [:ul
      (for [task tasks]
        [:li
          [:h3 (:title task)]
          [:p (:description task)]
          [:button {:onclick (str "editTask(" (:id task) ")")} "Editar"]
          [:button {:onclick (str "deleteTask(" (:id task) ")")} "Eliminar"]])]))
  1. Acceso a una base de datos en Luminus:
(defn create-task [task]
  (let [new-task (Task. (:title task) (:description task))]
    (db/save new-task)))

(defn update-task [id task]
  (db/update! (Task. (:title task) (:description task)) id))

(defn delete-task [id]
  (db/delete! (db/find-by-id Task id)))

Ejercicios prácticos con instrucciones claras¶

  1. Crear una aplicación web utilizando Luminus Framework que permita gestionar una lista de tareas.
  2. La aplicación debe tener las siguientes funcionalidades:
  3. Crear una nueva tarea con un título y una descripción.
  4. Editar una tarea existente.
  5. Eliminar una tarea.
  6. Marcar una tarea como completada.
  7. Utilizar una base de datos para almacenar las tareas.
  8. Implementar un sistema de autenticación de usuarios para que solo puedan acceder a la aplicación aquellos que estén registrados.
  9. Añadir validaciones en los campos de título y descripción para evitar tareas vacías o con información incompleta.

Consejos o mejores prácticas¶

  • Antes de comenzar a desarrollar, es importante planificar y definir claramente los objetivos y funcionalidades de la aplicación.
  • Utilizar la documentación oficial de Luminus para consultar dudas o encontrar soluciones a problemas.
  • Realizar pruebas continuas durante el desarrollo para asegurarse de que todo funciona correctamente.
  • Seguir buenas prácticas de programación, como utilizar nombres de variables descriptivos y comentarios adecuados en el código.
  • No tener miedo de probar nuevas funcionalidades o experimentar con diferentes enfoques para resolver un problema.
