Desarrollo de aplicaciones en tiempo real¶

En el mundo actual, la necesidad de aplicaciones en tiempo real se ha vuelto cada vez más relevante. Desde aplicaciones de mensajería hasta aplicaciones de monitoreo en tiempo real, la capacidad de actualizar y mostrar información en tiempo real se ha convertido en una característica esencial para muchas aplicaciones. En este módulo, se explicará cómo desarrollar aplicaciones en tiempo real utilizando Luminus Framework y otras herramientas y técnicas.

Teoría¶

Las aplicaciones en tiempo real se refieren a aquellas que actualizan y muestran información en tiempo real, sin la necesidad de que el usuario actualice o recargue la página. Esto se logra a través de tecnologías como WebSockets y Server-Sent Events (SSE). WebSockets permiten una comunicación bidireccional entre el cliente y el servidor, mientras que SSE permiten una comunicación unidireccional del servidor al cliente.

Al utilizar estas tecnologías en conjunto con Luminus Framework, es posible crear aplicaciones en tiempo real altamente eficientes y escalables. Luminus proporciona un conjunto de herramientas y bibliotecas que facilitan la implementación de estas tecnologías y la gestión de conexiones en tiempo real.

Palabras clave¶

  • Aplicaciones en tiempo real: son aquellas que actualizan y muestran información en tiempo real, sin la necesidad de que el usuario actualice o recargue la página.
  • WebSockets: es una tecnología que permite una comunicación bidireccional entre el cliente y el servidor.
  • Server-Sent Events (SSE): es una tecnología que permite una comunicación unidireccional del servidor al cliente.
  • Luminus Framework: es un framework para desarrollar aplicaciones web en Clojure.

Ejemplos de código en Luminus Framework¶

Implementación de WebSockets¶

En el archivo project.clj, se deben agregar las dependencias necesarias para utilizar WebSockets en Luminus:

:dependencies [[org.clojure/clojure "1.10.0"]
               [ring/ring-core "1.7.1"]
               [compojure "1.6.1"]
               [clj-websocket "0.4.1"]]

En el archivo handler.clj, se puede definir una ruta para la conexión WebSocket y un manejador para manejar los eventos de la conexión:

(ns myapp.handler
  (:require [ring.util.response :refer [response]]
            [clj-websocket.core :as ws]))

(defroutes app-routes
  (GET "/" [] (response "Hello World!"))
  (ws/websocket "/ws" [] ws-handler))

(defn ws-handler [session]
  (let [on-message (fn [msg]
                     (println "Recibido:" msg))]
    {:on-open (fn [ws]
                (println "Conexión abierta"))
     :on-message on-message
     :on-close (fn [ws]
                 (println "Conexión cerrada"))}))

Implementación de SSE¶

En el archivo project.clj, se deben agregar las dependencias necesarias para utilizar SSE en Luminus:

:dependencies [[org.clojure/clojure "1.10.0"]
               [ring/ring-core "1.7.1"]
               [compojure "1.6.1"]
               [ring/ring-sse "0.2.0"]]

En el archivo handler.clj, se puede definir una ruta para la conexión SSE y un manejador para enviar eventos al cliente:

(ns myapp.handler
  (:require [ring.util.response :refer [response]]
            [ring.sse :refer [event]]
            [clojure.core.async :refer [chan put!]])
  (:import [java.util.concurrent Executors]))

(defroutes app-routes
  (GET "/" [] (response "Hello World!"))
  (GET "/sse" [] sse-handler))

(defn sse-handler [request]
  (let [sse-channel (chan)]
    (Executors/ newSingleThreadExecutor)
    (.submit (fn []
               (put! sse-channel (event "message" "Hola!"))
               (put! sse-channel (event "message" "¿Cómo estás?"))))
    {:status 200
     :headers {"Content-Type" "text/event-stream"}
     :body (ring.sse/stream sse-channel)}))

  1. Crea una aplicación en Luminus que utilice WebSockets para mostrar en tiempo real la temperatura de una ciudad.
  2. Modifica la aplicación anterior para que utilice SSE en lugar de WebSockets.
  3. Implementa un chat en tiempo real utilizando WebSockets y Luminus.

Consejos y mejores prácticas¶

  • Utiliza WebSockets para aplicaciones que requieren una comunicación bidireccional entre el cliente y el servidor, como chats o juegos en línea.
  • Utiliza SSE para aplicaciones que solo requieren una comunicación unidireccional del servidor al cliente, como monitoreo en tiempo real.
  • Utiliza bibliotecas y herramientas disponibles en Luminus para simplificar la implementación de WebSockets y SSE.
  • Asegúrate de optimizar el rendimiento de tu aplicación en tiempo real, ya que el uso de estas tecnologías puede afectar la velocidad de carga de tu aplicación.
