Proyecto final¶

Descripción del módulo¶

En este módulo se realizará un proyecto final en el que se aplicarán todos los conocimientos adquiridos durante el curso de Luminus Framework. El objetivo principal es que los estudiantes puedan poner en práctica lo aprendido y desarrollar una aplicación más compleja utilizando este framework. Al final del curso, se presentará el proyecto final y se evaluará la comprensión y aplicación de los conceptos.

Explicación teórica¶

El proyecto final es una forma de consolidar los conocimientos adquiridos a lo largo del curso de Luminus Framework. En este módulo, los estudiantes tendrán la oportunidad de desarrollar una aplicación completa, utilizando todas las funcionalidades del framework y aplicando buenas prácticas de programación. Además, también podrán demostrar su creatividad e innovación en el diseño y funcionalidad de la aplicación.

Palabras clave y su definición¶

  • Luminus Framework: Framework de desarrollo web basado en Clojure y que utiliza herramientas como Ring, Compojure y ClojureScript para crear aplicaciones web.
  • Aplicación web: Es un programa informático que se ejecuta en un servidor web y que permite al usuario interactuar a través de un navegador web.
  • Ring: Biblioteca de Clojure que proporciona una interfaz unificada para las aplicaciones web.
  • Compojure: Biblioteca de Clojure que proporciona un enrutador para las aplicaciones web.
  • ClojureScript: Dialecto de Clojure que compila a JavaScript y permite desarrollar aplicaciones para el lado del cliente.

Ejercicios prácticos¶

  1. Desarrollar una aplicación web utilizando Luminus Framework.
  2. Implementar un enrutador utilizando Compojure.
  3. Crear una página de inicio utilizando la función home-page del ejemplo de código.
  4. Agregar una funcionalidad de autenticación utilizando Ring.
  5. Desplegar la aplicación en un servidor web.

Consejos o mejores prácticas¶

  • Planificar bien el proyecto antes de comenzar a escribir código.
  • Utilizar buenas prácticas de programación y seguir las convenciones de Luminus Framework.
  • Revisar y probar el código regularmente para evitar errores.
  • Documentar el código para facilitar su comprensión y mantenimiento.
  • Utilizar herramientas como Leiningen para gestionar las dependencias y el despliegue de la aplicación.
