Middlewares y Plug en Phoenix Framework¶

Explicación teórica¶

Los middlewares y Plug son dos conceptos fundamentales en el desarrollo de aplicaciones web con Phoenix Framework. Ambos se utilizan para agregar funcionalidades a nuestras aplicaciones de forma modular y reutilizable.

Middlewares¶

Los middlewares son funciones que se ejecutan antes o después de que una solicitud llegue a nuestra aplicación. Son capas intermedias entre el servidor web y nuestra aplicación, y nos permiten realizar tareas comunes a todas las solicitudes, como autenticación, registro de errores, entre otros.

En Phoenix, los middlewares se definen en el archivo endpoint.ex, que actúa como punto de entrada de todas las solicitudes. Allí se pueden agregar, modificar o eliminar middlewares según las necesidades de nuestra aplicación.

Plug¶

Plug es un módulo de Phoenix que nos permite crear nuestros propios middlewares. Se basa en el concepto de "enchufes" (plugs en inglés), que son funciones que reciben una solicitud y devuelven una respuesta, y pueden ser encadenados para realizar una serie de tareas en secuencia.

Con Plug, podemos definir middlewares personalizados para agregar funcionalidades específicas a nuestra aplicación. Además, también podemos utilizar los middlewares ya incluidos en el framework, como por ejemplo Plug.Logger para registrar las solicitudes en el servidor.

Palabras clave y su definición¶

  • Middlewares: Funciones que se ejecutan antes o después de una solicitud en una aplicación web.
  • Plug: Módulo de Phoenix que nos permite crear middlewares personalizados.
  • Endpoint: Archivo de configuración en Phoenix donde se definen los middlewares.
  • Enchufes: Funciones que reciben una solicitud y devuelven una respuesta, utilizados en Plug para crear middlewares.
  • Encadenar: Proceso de conectar varios middlewares en secuencia para realizar una serie de tareas.

Ejemplos de código en Phoenix Framework¶

Definir un middleware en endpoint.ex¶

defmodule MyApp.Endpoint do
  use Phoenix.Endpoint, otp_app: :my_app

  # Se define el middleware utilizando el módulo Plug.Logger
  plug Plug.Logger

  # Resto de la configuración del endpoint
  # ...
end

Crear un middleware personalizado con Plug¶

defmodule MyApp.CustomMiddleware do
  # El enchufe recibe la solicitud y devuelve la misma solicitud modificada
  def init(opts), do: opts

  def call(conn, _opts) do
    # Realizar tareas adicionales con la solicitud
    # ...

    # Devolver la solicitud modificada
    conn
  end
end

Encadenar varios middlewares con Plug¶

defmodule MyApp.Endpoint do
  use Phoenix.Endpoint, otp_app: :my_app

  # Se encadenan varios middlewares utilizando el operador `|>`
  plug Plug.Logger
  |> plug MyApp.CustomMiddleware
  |> plug Plug.Session

  # Resto de la configuración del endpoint
  # ...
end

Ejercicios prácticos con instrucciones claras¶

  1. Crea un middleware personalizado que agregue un encabezado X-App-Version a todas las respuestas de tu aplicación.
  2. Encadena el middleware creado en el ejercicio anterior con el middleware Plug.Static para servir archivos estáticos solo si la solicitud contiene el encabezado X-App-Version.
  3. Agrega un middleware que redireccione todas las solicitudes a una página de mantenimiento si la aplicación se encuentra en modo de mantenimiento (definido en una configuración global).
  4. Crea un middleware que registre en una base de datos todas las solicitudes que produzcan un error 500 en el servidor.

Consejos o mejores prácticas¶

  • Utiliza middlewares para realizar tareas comunes en todas las solicitudes, como autenticación o registro de errores.
  • Crea middlewares personalizados con Plug para agregar funcionalidades específicas a tu aplicación.
  • Encadena middlewares en el orden correcto para realizar tareas en secuencia.
  • Utiliza la documentación de Plug para conocer más sobre los enchufes disponibles y cómo crear tus propios middlewares.
  • No sobrecargues tu aplicación con demasiados middlewares, ya que pueden afectar el rendimiento. Utilízalos solo cuando sea necesario.
