Pruebas en Phoenix¶

Descripción del módulo¶

En este módulo, aprenderemos sobre la importancia de realizar pruebas automatizadas en nuestra aplicación Phoenix y cómo utilizar herramientas como ExUnit y Hound para facilitar este proceso. Las pruebas son una parte esencial del desarrollo de software, ya que nos permiten asegurarnos de que nuestro código funciona correctamente y nos ayudan a detectar y corregir errores antes de que lleguen a producción.

Explicación teórica¶

Las pruebas automatizadas son una técnica de desarrollo de software en la que se utilizan herramientas y scripts para ejecutar pruebas en el código de manera automática. En Phoenix, estas pruebas se pueden realizar utilizando la herramienta integrada ExUnit, que nos permite escribir pruebas para nuestros controladores, modelos, vistas y otras partes de nuestra aplicación.

Existen varios tipos de pruebas que se pueden realizar en Phoenix, entre las más comunes se encuentran las pruebas unitarias, que se enfocan en probar unidades individuales de código, y las pruebas de integración, que se utilizan para probar la interacción entre diferentes partes de la aplicación. Además, también podemos utilizar herramientas como Hound para realizar pruebas de aceptación, que simulan interacciones de usuario en nuestra aplicación.

Palabras clave y su definición¶

  • Pruebas automatizadas: técnica de desarrollo de software en la que se utilizan herramientas y scripts para ejecutar pruebas en el código de manera automática.
  • ExUnit: herramienta de pruebas integrada en Phoenix que nos permite escribir y ejecutar pruebas en nuestro código.
  • Pruebas unitarias: tipo de prueba que se enfoca en probar unidades individuales de código.
  • Pruebas de integración: tipo de prueba que se utiliza para probar la interacción entre diferentes partes de la aplicación.
  • Hound: herramienta de pruebas de aceptación en Phoenix que simula interacciones de usuario en nuestra aplicación.

Ejemplos de código en Phoenix Framework¶

Pruebas unitarias¶

defmodule MyApp.UserTest do
  use MyApp.DataCase

  test "creates a new user" do
    user = %User{name: "John Doe", email: "john@doe.com", password: "password"}
    assert {:ok, %User{}} = MyApp.User.create_user(user)
  end
end

Pruebas de integración¶

defmodule MyApp.PostControllerTest do
  use MyApp.ConnCase

  test "returns a list of posts" do
    conn = get conn, "/api/posts"
    assert json_response(conn, 200) == %{posts: []}
  end
end

Pruebas de aceptación con Hound¶

defmodule MyApp.RegistrationTest do
  use MyApp.AcceptanceCase

  test "registers a new user" do
    visit "/register"

    fill_in "Name", with: "John Doe"
    fill_in "Email", with: "john@doe.com"
    fill_in "Password", with: "password"
    click_button "Register"

    assert page_has_content?("Welcome, John!")
  end
end

Ejercicios prácticos con instrucciones claras¶

  1. Crea una prueba unitaria que verifique que un usuario se pueda autenticar correctamente en tu aplicación Phoenix.
  2. Escribe una prueba de integración que verifique que se puedan crear nuevos posts en tu aplicación.
  3. Utilizando Hound, realiza una prueba de aceptación que simule el proceso de registro de un nuevo usuario en tu aplicación.

Consejos o mejores prácticas¶

  • Es importante tener una buena cobertura de pruebas en nuestra aplicación, ya que esto nos ayudará a detectar y corregir errores de manera más eficiente.
  • Utiliza nombres descriptivos para tus pruebas, para que sea más fácil entender qué se está probando.
  • Asegúrate de incluir tanto pruebas positivas como negativas en tus suites de pruebas.
  • Utiliza fixtures o factories para crear datos de prueba en lugar de utilizar datos reales de la base de datos.
  • Realiza pruebas de manera regular y no solo al final del desarrollo, esto te ayudará a detectar errores más rápido.
