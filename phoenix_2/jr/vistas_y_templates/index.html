Vistas y templates en Phoenix Framework¶

Explicación teórica¶

En Phoenix Framework, las vistas y templates son elementos fundamentales para la creación de aplicaciones web. Una vista es una representación visual de los datos que se deben mostrar en una página web, mientras que un template es un archivo que define la estructura y el contenido de una vista.

En esencia, las vistas y templates son los encargados de convertir los datos que se generan en el backend en una página web que pueda ser visualizada por el usuario. La separación de la lógica de negocio y la presentación es una de las principales ventajas de utilizar vistas y templates en una aplicación web, ya que permite un desarrollo más modular y mantenible.

Palabras clave y su definición¶

  • Vista: Representación visual de los datos que se muestran en una página web.
  • Template: Archivo que define la estructura y contenido de una vista.
  • Renderizar: Proceso de convertir datos en una página web para su visualización.
  • Layout: Plantilla base que define la estructura común de todas las vistas de una aplicación web.
  • Partial: Porción de código HTML reutilizable que se puede incluir en diferentes vistas.
  • EEx: Embedded Elixir, lenguaje de plantillas utilizado en Phoenix Framework.
  • Plugs: Funciones que se ejecutan antes o después de la renderización de una vista para realizar tareas adicionales.
  • Helpers: Funciones auxiliares que se pueden utilizar en las vistas para simplificar su código.

Ejemplos de código en Phoenix Framework¶

Ejemplo de una vista básica en Phoenix Framework:

defmodule MyApp.HomeView do
  use MyApp.Web, :view

  def render("index.html", assigns) do
    ~H"""
    <html>
      <head>
        <title>Bienvenido a mi aplicación</title>
      </head>
      <body>
        <h1>Hola <%= assigns[:name] %></h1>
      </body>
    </html>
    """
  end
end

Ejemplo de un template en Phoenix Framework:

<!DOCTYPE html>
<html>
  <head>
    <title><%= @title %> - Mi aplicación</title>
  </head>
  <body>
    <%= render @view_module, @view_template, assigns %>
  </body>
</html>

Ejercicios prácticos con instrucciones claras¶

  1. Crea una vista llamada "about" que muestre información sobre tu aplicación.
  2. Crea un template llamado "default" que sirva como layout base para todas las vistas de tu aplicación.
  3. Utiliza un partial para mostrar un menú de navegación en todas las vistas de tu aplicación.
  4. Crea un helper que formatee una fecha en un formato específico y úsalo en una vista.
  5. Crea un plug que redirija a los usuarios a la página de inicio si intentan acceder a una vista sin estar autenticados.

Consejos o mejores prácticas¶

  • Utiliza un layout base para todas las vistas de tu aplicación para mantener una estructura consistente.
  • Divide tus vistas en archivos separados para una mayor organización.
  • Utiliza partials para reutilizar código HTML en diferentes vistas.
  • Utiliza helpers para simplificar el código de tus vistas y evitar la repetición.
  • Utiliza plugs para ejecutar tareas adicionales antes o después de la renderización de una vista.
  • Utiliza EEx para escribir plantillas más legibles y dinámicas.
  • Utiliza la convención de nombres de Phoenix para tus vistas y templates (ej: "home_view.ex" y "home.html.eex").
  • Utiliza los módulos de ayuda en Phoenix para obtener más información sobre cómo trabajar con vistas y templates.
