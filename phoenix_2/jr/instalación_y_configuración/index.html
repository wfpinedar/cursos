Instalación y configuración de Phoenix Framework¶

Explicación teórica¶

Phoenix Framework es un framework de desarrollo web para el lenguaje de programación Elixir. Fue lanzado en 2015 y se basa en el modelo de arquitectura MVC (Modelo-Vista-Controlador) para el desarrollo de aplicaciones web. Phoenix se destaca por ser rápido, escalable y por su capacidad para manejar grandes cantidades de tráfico en tiempo real.

Para poder utilizar Phoenix Framework, es necesario instalar y configurar algunas herramientas como Elixir, Erlang y Node.js. Una vez que estos requisitos estén satisfechos, se puede proceder con la instalación de Phoenix.

Palabras clave y su definición¶

  • Elixir: Es un lenguaje de programación funcional creado en 2012 por José Valim. Se ejecuta sobre la máquina virtual de Erlang (BEAM) y se basa en el paradigma de programación funcional y concurrencia.
  • Erlang: Es una plataforma de desarrollo creada por Ericsson en 1986. Es altamente escalable y está diseñada para manejar aplicaciones distribuidas y concurrentes.
  • Node.js: Es un entorno de ejecución de JavaScript construido sobre el motor V8 de Google Chrome. Se utiliza principalmente para crear aplicaciones de red y servidores web.
  • MVC: Es un patrón de arquitectura de software que separa la aplicación en tres componentes principales: modelo, vista y controlador. El modelo se encarga de la lógica de negocio y la persistencia de datos, la vista muestra la información al usuario y el controlador maneja las solicitudes del usuario y coordina la comunicación entre el modelo y la vista.

Ejemplos de código en Phoenix Framework¶

A continuación, se muestran ejemplos de código en Phoenix Framework para crear una aplicación simple y mostrar un mensaje de bienvenida.

Creación de una aplicación Phoenix¶

Para crear una aplicación Phoenix, se utiliza el comando mix phx.new <nombre de la aplicación>. Esto generará una estructura de archivos para la aplicación, incluyendo el archivo mix.exs que contiene las dependencias y la configuración de la aplicación.

Definición de una ruta y controlador¶

En el archivo router.ex, se definen las rutas de la aplicación. Por ejemplo, para crear una ruta que muestre un mensaje de bienvenida, se puede agregar el siguiente código:

get "/", PageController, :index

Esto indica que cuando se recibe una solicitud GET en la ruta "/", se debe llamar al controlador PageController y al método index.

Creación de un controlador¶

En el archivo page_controller.ex, se define el controlador PageController y el método index que devuelve el mensaje de bienvenida:

defmodule MyApp.PageController do
  use MyApp, :controller

  def index(conn, _params) do
    render(conn, "index.html", message: "¡Bienvenido a mi aplicación Phoenix!")
  end
end

Creación de una vista¶

En el archivo index.html.eex, se define la vista que mostrará el mensaje de bienvenida:

<h1><%= @message %></h1>

Iniciar el servidor¶

Para iniciar el servidor y ver la aplicación en el navegador, se utiliza el comando mix phx.server.

Ejercicios prácticos con instrucciones claras¶

  1. Instala Elixir, Erlang y Node.js en tu sistema.
  2. Crea una aplicación Phoenix utilizando el comando mix phx.new.
  3. Define una ruta y un controlador que muestren un mensaje de bienvenida.
  4. Crea una vista para mostrar el mensaje.
  5. Inicia el servidor y verifica que la aplicación funcione correctamente en el navegador.

Consejos o mejores prácticas¶

  • Es recomendable seguir la estructura de archivos y convenciones de nomenclatura utilizadas por Phoenix Framework para una mejor organización y legibilidad del código.
  • Utiliza el archivo mix.exs para agregar dependencias y configurar tu aplicación.
  • Asegúrate de tener conocimientos básicos de Elixir y su sintaxis antes de comenzar a trabajar con Phoenix.
  • Utiliza el módulo Plug para manejar las solicitudes HTTP y para crear middleware personalizado.
  • Comienza con una aplicación simple y ve agregando funcionalidades a medida que te familiarices con el framework.
