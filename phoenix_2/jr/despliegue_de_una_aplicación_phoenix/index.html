Despliegue de una aplicación Phoenix¶

El despliegue de una aplicación Phoenix en un servidor de producción es un proceso clave para que la aplicación pueda estar disponible para su uso en línea. En este módulo, aprenderemos los pasos necesarios para realizar un despliegue exitoso de una aplicación Phoenix en un entorno de producción.

Explicación teórica¶

Antes de comenzar con el proceso de despliegue, es importante comprender los conceptos básicos detrás de una aplicación Phoenix y su estructura. Phoenix es un framework web escrito en el lenguaje de programación Elixir, que se basa en el patrón de arquitectura MVC (Modelo-Vista-Controlador). Esto significa que la aplicación se divide en tres capas: el modelo, que se encarga de la lógica de negocio; la vista, que se encarga de la presentación de los datos; y el controlador, que actúa como intermediario entre el modelo y la vista.

La estructura de una aplicación Phoenix consta de varios directorios y archivos, cada uno con un propósito específico. El directorio lib contiene el código fuente de la aplicación, mientras que el directorio test contiene las pruebas automatizadas. El archivo mix.exs es el encargado de manejar las dependencias y la configuración de la aplicación. Por último, el archivo config.exs contiene la configuración de la aplicación, como la base de datos y las rutas de la aplicación.

Palabras clave y su definición¶

  • Phoenix: Framework web escrito en el lenguaje de programación Elixir.
  • Elixir: Lenguaje de programación funcional basado en Erlang.
  • MVC: Patrón de arquitectura Modelo-Vista-Controlador.
  • Modelo: Capa encargada de la lógica de negocio.
  • Vista: Capa encargada de la presentación de los datos.
  • Controlador: Capa encargada de actuar como intermediario entre el modelo y la vista.
  • mix.exs: Archivo encargado de manejar las dependencias y la configuración de la aplicación.
  • config.exs: Archivo encargado de contener la configuración de la aplicación.

Ejemplos de código en Phoenix Framework¶

A continuación, se muestran algunos ejemplos de código en Phoenix que son relevantes para el proceso de despliegue de una aplicación:

  1. Crear una nueva aplicación Phoenix con el comando mix phx.new <nombre de la aplicación>.
  2. Configurar la base de datos en el archivo config.exs:
config :my_app, MyApp.Repo,
  username: "username",
  password: "password",
  database: "my_app_prod",
  hostname: "localhost",
  pool_size: 20
  1. Generar una migración de base de datos con el comando mix ecto.gen.migration <nombre de la migración>.
  2. Ejecutar la migración con el comando mix ecto.migrate.
  3. Crear una nueva ruta en el archivo router.ex:
get "/", PageController, :index
  1. Crear un controlador para la ruta con el comando mix phx.gen.html <nombre del controlador> <nombre del modelo> <campos del modelo>.
  2. Configurar la aplicación para que se ejecute en modo producción en el archivo config/prod.exs:
config :my_app, MyApp.Endpoint,
  url: [scheme: "https", host: "myapp.com", port: 443],
  cache_static_manifest: "priv/static/cache_manifest.json",
  secret_key_base: System.get_env("SECRET_KEY_BASE")

Ejercicios prácticos con instrucciones claras¶

  1. Crear una nueva aplicación Phoenix con el nombre de "my_app".
  2. Configurar la base de datos en el archivo config.exs con los siguientes valores: username: "admin", password: "123456", database: "my_app_prod", hostname: "localhost", pool_size: 10.
  3. Generar una migración de base de datos con el nombre de "create_users" que contenga los campos "name", "email" y "password".
  4. Ejecutar la migración.
  5. Crear una nueva ruta en el archivo router.ex que apunte a la acción "index" del controlador "PageController".
  6. Crear un controlador con el nombre de "PageController" y la acción "index".
  7. Configurar la aplicación para que se ejecute en modo producción en el archivo config/prod.exs con la url "https://myapp.com" y una clave secreta generada.
  8. Ejecutar la aplicación en modo producción con el comando MIX_ENV=prod mix phx.server.
  9. Probar la aplicación en línea en la dirección "https://myapp.com".

Consejos o mejores prácticas¶

  1. Realizar pruebas exhaustivas antes de realizar el despliegue en un servidor de producción.
  2. Utilizar un sistema de control de versiones para mantener un historial de los cambios en la aplicación.
  3. Asegurarse de tener una copia de seguridad de la base de datos antes de realizar el despliegue.
  4. Utilizar herramientas de automatización de despliegue, como Capistrano o edeliver, para facilitar el proceso.
  5. Configurar un sistema de monitoreo para detectar cualquier problema en la aplicación en tiempo real.
  6. Realizar actualizaciones de forma regular para mantener la seguridad y el rendimiento de la aplicación.
  7. Documentar el proceso de despliegue para facilitar futuras actualizaciones o para otros miembros del equipo que puedan realizar el despliegue en el futuro.
