Internals de Phoenix¶

Introducción¶

Phoenix Framework es un framework web escrito en el lenguaje de programación Elixir, que se basa en el patrón de arquitectura Modelo-Vista-Controlador (MVC). Fue creado para ser rápido, escalable y confiable, y está desarrollado por el equipo de Elixir de la empresa de consultoría tecnológica Plataformatec.

En esta lección, profundizaremos en los entresijos de Phoenix, incluyendo su arquitectura interna y cómo funciona el enrutamiento. Aprenderemos sobre la estructura de un proyecto Phoenix, cómo se manejan las solicitudes HTTP y cómo se realiza el enrutamiento de las mismas. También cubriremos algunas palabras clave importantes y consejos para aprovechar al máximo Phoenix.

Estructura de un proyecto Phoenix¶

Un proyecto Phoenix sigue una estructura de directorios y archivos común, que es similar a otros frameworks web como Ruby on Rails. Al crear un nuevo proyecto Phoenix, se generan automáticamente los siguientes directorios:

  • config: contiene la configuración del proyecto, como las credenciales de la base de datos y la configuración del servidor web.
  • lib: contiene el código fuente del proyecto, incluyendo los modelos, controladores y vistas.
  • priv: contiene recursos privados del proyecto, como los archivos estáticos y las plantillas HTML.
  • test: contiene los casos de prueba del proyecto.
  • web: contiene los archivos relacionados con la interfaz web del proyecto, incluyendo el enrutador y los controladores.

Enrutamiento en Phoenix¶

Phoenix utiliza un enrutador para manejar las solicitudes HTTP entrantes y dirigirlas a los controladores correspondientes. El enrutador se encuentra en el directorio web y se llama router.ex. Este archivo contiene una función scope que define la ruta base para todas las solicitudes entrantes. Dentro de esta función, se pueden definir rutas específicas utilizando el método get o post, que especifican el método HTTP correspondiente, la ruta y el controlador al que se debe dirigir.

Por ejemplo, si tenemos una solicitud GET a la ruta /hola, el enrutador buscará una función hola en el controlador correspondiente y ejecutará el código dentro de ella.

Palabras clave importantes¶

  • Controlador: un módulo que maneja las solicitudes HTTP y responde con una vista o una API.
  • Vista: un módulo que genera una respuesta HTTP, ya sea una página HTML o un JSON.
  • Modelo: un módulo que representa una entidad de la base de datos y maneja la lógica de negocio relacionada con ella.
  • Enrutador: un módulo que maneja las solicitudes entrantes y las dirige al controlador correspondiente.
  • Plug: una función que se ejecuta antes o después de una solicitud y permite realizar operaciones de middleware, como autenticación o compresión de respuestas.

Ejemplos de código en Phoenix Framework¶

  1. Definir una ruta en el enrutador:
get "/usuarios", UserController, :index
  1. Crear una función en el controlador para manejar la solicitud:
def index(conn, _params) do
  usuarios = Repo.all(Usuario)
  render(conn, "index.html", usuarios: usuarios)
end
  1. Crear una vista para generar una respuesta HTML:
defmodule MyApp.UsuarioView do
  use MyApp.Web, :view

  def render("index.html", %{usuarios: usuarios}) do
    # Generar una lista de usuarios
    usuarios_html = for usuario <- usuarios do
      "<li>#{usuario.nombre}</li>"
    end

    # Renderizar la página con la lista de usuarios
    render_layout("app.html", usuarios_html: usuarios_html)
  end
end

Ejercicios prácticos¶

  1. Crea una nueva ruta en el enrutador que maneje una solicitud POST a la ruta /usuarios y la dirija al controlador correspondiente.
  2. En el controlador, crea una función para manejar la solicitud y agregar un nuevo usuario a la base de datos utilizando el modelo correspondiente.
  3. En la vista, genera una respuesta de éxito si el usuario se ha creado correctamente o una respuesta de error si ha fallado.

Consejos y mejores prácticas¶

  • Utiliza los recursos de la comunidad de Elixir y Phoenix, como la documentación oficial, foros y grupos de discusión, para obtener ayuda y aprender de otros desarrolladores.
  • Sigue los principios de diseño web y los patrones de arquitectura MVC para mantener un código limpio y escalable.
  • Utiliza los plugs para realizar tareas comunes como autenticación y manejo de errores en lugar de repetir código en los controladores.
  • Aprovecha las características de Elixir, como el enrutamiento de patrones y la concurrencia, para mejorar el rendimiento de tu aplicación.
  • Practica y experimenta con diferentes aspectos de Phoenix para familiarizarte con su funcionamiento y mejorar tus habilidades como desarrollador.
