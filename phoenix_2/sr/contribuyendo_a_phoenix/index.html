Contribuyendo a Phoenix¶

Descripción del módulo¶

Phoenix es un framework de desarrollo web escrito en el lenguaje de programación Elixir, basado en el patrón de arquitectura MVC (Modelo-Vista-Controlador). Fue creado para ser rápido, escalable y confiable, y se ha convertido en una de las herramientas más populares para el desarrollo de aplicaciones web en Elixir.

Este módulo se enfoca en cómo contribuir al desarrollo de Phoenix y colaborar con la comunidad para mejorar la plataforma. Aprenderás cómo contribuir al código fuente de Phoenix, cómo informar de errores y cómo colaborar con otros desarrolladores en la comunidad de Phoenix.

Palabras clave y definiciones¶

  • Contribución: aportar al desarrollo de una plataforma o proyecto de manera activa y voluntaria.
  • Código fuente: el conjunto de instrucciones escritas en un lenguaje de programación que conforman un programa o aplicación.
  • Error: un problema o fallo en el funcionamiento de una aplicación.
  • Comunidad: grupo de personas que comparten intereses y trabajan juntas en un proyecto o causa común.
  • Pull request: una solicitud para que los cambios realizados en el código fuente sean incorporados al repositorio principal.
  • Fork: una copia del repositorio de código fuente de un proyecto que se puede modificar y utilizar de manera independiente.

Ejemplos de código en Phoenix Framework¶

Instalación de Phoenix¶

Para contribuir a Phoenix, primero debes tenerlo instalado en tu sistema. Puedes seguir los siguientes pasos para instalar Phoenix en una máquina con Linux o Mac:

  1. Instala Elixir en tu sistema.
  2. Instala Hex, el administrador de paquetes de Elixir, ejecutando mix local.hex en tu terminal.
  3. Instala Phoenix con el comando mix archive.install hex phx_new.
  4. Verifica que la instalación fue exitosa con phx_new -v.

Creación de una nueva aplicación¶

Una vez que tengas Phoenix instalado, puedes crear una nueva aplicación para empezar a contribuir. Puedes hacerlo con el siguiente comando:

mix phx.new my_app

Esto creará una nueva aplicación llamada my_app en el directorio actual.

Agregando una nueva funcionalidad¶

Supongamos que queremos agregar una nueva funcionalidad a nuestra aplicación que permita a los usuarios autenticarse con Google. Podemos hacerlo siguiendo estos pasos:

  1. Instalar la biblioteca oauth2 con mix deps.get.
  2. Crear un controlador AuthController con el comando mix phx.gen.controller Auth auth google (reemplaza google con el proveedor de autenticación que quieras).
  3. Agregar la lógica de autenticación en el controlador y las rutas correspondientes en el archivo router.ex.
  4. Actualizar las vistas necesarias para mostrar los botones de autenticación.
  5. Modificar el modelo User para almacenar la información del usuario autenticado.
  6. Realizar pruebas para asegurarse de que la funcionalidad funciona correctamente.

Ejercicios prácticos con instrucciones claras¶

  1. Crea un fork del repositorio de Phoenix en GitHub.
  2. Realiza los cambios que consideres necesarios en tu fork.
  3. Crea un pull request con tus cambios en el repositorio principal.
  4. Contribuye a la documentación de Phoenix en el repositorio oficial o en la wiki de la comunidad.
  5. Busca y reporta errores en la aplicación de ejemplo de Phoenix (disponible en la documentación oficial).
  6. Participa en discusiones y debates en el foro de la comunidad de Phoenix.

Consejos o mejores prácticas¶

  • Antes de contribuir, asegúrate de leer las pautas y recomendaciones de contribución del proyecto.
  • Siempre incluye pruebas para tus cambios y asegúrate de que todas las pruebas existentes pasen correctamente.
  • Utiliza comentarios y documentación clara y concisa en tu código para facilitar la revisión de tu contribución.
  • Participa en la comunidad de Phoenix, comparte tus conocimientos y aprende de otros desarrolladores.
  • No tengas miedo de pedir ayuda o hacer preguntas si tienes dudas o problemas durante el proceso de contribución. La comunidad de Phoenix es muy amigable y siempre está dispuesta a ayudar.
